# 从零实现自注意力（Self-Attention）

> 注意力是一张查找表，每个词都在问"谁对我重要？"——然后学会答案。

**类型：** 构建
**语言：** Python
**前置要求：** Phase 3（深度学习核心）、Phase 5 第10课（序列到序列）
**时间：** 约90分钟

## 学习目标

- 仅用 NumPy 从零实现缩放点积自注意力，包括查询/键/值投影和 softmax 加权求和
- 构建多头注意力层，实现头分拆、并行注意力计算和结果拼接
- 追踪注意力矩阵如何捕获令牌关系，并解释为什么除以 sqrt(dk) 能防止 softmax 饱和
- 应用因果掩码将双向注意力转换为自回归（解码器式）注意力

## 问题所在

RNN 一次处理一个令牌。当处理到第50个令牌时，来自第1个令牌的信息已被挤压经过50次压缩步骤。长距离依赖被压入固定大小的隐藏状态——这是LSTM门控机制无法完全解决的瓶颈。

2014年 Bahdanau 注意力论文给出了修复方案：让解码器回看每个编码器位置，决定哪些对当前步骤重要。但它仍然附加在RNN之上。2017年的《Attention Is All You Need》论文提出了更尖锐的问题：如果注意力是*唯一*的机制呢？没有循环。没有卷积。只有注意力。

自注意力让序列中的每个位置在单个并行步骤中关注所有其他位置。这正是Transformer快速、可扩展且占据主导地位的原因。

## 核心概念

### 数据库查找类比

把注意力想象成软性数据库查找：

```
传统数据库：
  查询："法国首都"  -->  精确匹配  -->  "巴黎"

注意力：
  查询："法国首都"  -->  与所有键的相似度  -->  所有值的加权混合
```

每个令牌生成三个向量：
- **查询（Query, Q）**："我在找什么？"
- **键（Key, K）**："我包含什么？"
- **值（Value, V）**："如果被选中，我提供什么信息？"

查询与所有键之间的点积产生注意力分数。高分意味着"这个键匹配我的查询"。这些分数对值进行加权。输出是值的加权求和。

### Q、K、V 的计算

每个令牌嵌入通过三个可学习的权重矩阵进行投影：

```
输入嵌入（n 个令牌的序列，每个 d 维）：

  X = [x1, x2, x3, ..., xn]       形状: (n, d)

三个权重矩阵：

  Wq  形状: (d, dk)
  Wk  形状: (d, dk)
  Wv  形状: (d, dv)

投影：

  Q = X @ Wq    形状: (n, dk)      每个令牌的查询
  K = X @ Wk    形状: (n, dk)      每个令牌的键
  V = X @ Wv    形状: (n, dv)      每个令牌的值
```

可视化，单个令牌的处理流程：

```
             Wq
  x_i ------[*]------> q_i    "我在找什么？"
       |
       |     Wk
       +----[*]------> k_i    "我包含什么？"
       |
       |     Wv
       +----[*]------> v_i    "我提供什么？"
```

### 注意力矩阵

一旦得到所有令牌的 Q、K、V，注意力分数形成一个矩阵：

```
Scores = Q @ K^T    形状: (n, n)

              k1    k2    k3    k4    k5
        +-----+-----+-----+-----+-----+
   q1   | 2.1 | 0.3 | 0.1 | 0.8 | 0.2 |   <- q1 对每个键的关注程度
        +-----+-----+-----+-----+-----+
   q2   | 0.4 | 1.9 | 0.7 | 0.1 | 0.3 |
        +-----+-----+-----+-----+-----+
   q3   | 0.2 | 0.6 | 2.3 | 0.5 | 0.1 |
        +-----+-----+-----+-----+-----+
   q4   | 0.9 | 0.1 | 0.4 | 1.7 | 0.6 |
        +-----+-----+-----+-----+-----+
   q5   | 0.1 | 0.3 | 0.2 | 0.5 | 2.0 |
        +-----+-----+-----+-----+-----+

每行：一个令牌对整条序列的注意力分布
```

### 为什么要缩放？

点积随维度 dk 增长。如果 dk = 64，点积可能达到数十的量级，将 softmax 推入梯度消失区域。修复方法：除以 sqrt(dk)。

```
缩放后的分数 = (Q @ K^T) / sqrt(dk)
```

这使值保持在 softmax 能产生有用梯度的范围内。

### Softmax 将分数转为权重

Softmax 将原始分数转换为每行的概率分布：

```
q1 的原始分数：     [2.1, 0.3, 0.1, 0.8, 0.2]
                            |
                         softmax
                            |
注意力权重：         [0.52, 0.09, 0.07, 0.14, 0.08]   （总和约为1.0）
```

现在每个令牌都有一组权重，表示对每个其他令牌的关注程度。

### 值的加权求和

每个令牌的最终输出是所有值向量的加权求和：

```
output_i = sum( attention_weight[i][j] * v_j  对所有 j 求和 )

对于令牌 1：
  output_1 = 0.52 * v1 + 0.09 * v2 + 0.07 * v3 + 0.14 * v4 + 0.08 * v5
```

### 完整流程

```
                    +-------+
  X（输入）  ----->|  @ Wq  |-----> Q
                    +-------+
                    +-------+
  X（输入）  ----->|  @ Wk  |-----> K
                    +-------+                     +----------+
                    +-------+                     |          |
  X（输入）  ----->|  @ Wv  |-----> V ---------->| 加权求和 |----> 输出
                    +-------+          ^          |          |
                                       |          +----------+
                              +--------+--------+
                              |    softmax      |
                              +---------+-------+
                                        ^
                              +---------+-------+
                              | Q @ K^T / sqrt  |
                              +-----------------+
```

一行公式：

```
Attention(Q, K, V) = softmax( Q @ K^T / sqrt(dk) ) @ V
```

## 动手构建

### 步骤 1：从零实现 Softmax

Softmax 将原始 logit 转换为概率。减去最大值以确保数值稳定性。

```python
import numpy as np

def softmax(x):
    shifted = x - np.max(x, axis=-1, keepdims=True)  # 减去最大值，保证数值稳定
    exp_x = np.exp(shifted)
    return exp_x / np.sum(exp_x, axis=-1, keepdims=True)

logits = np.array([2.0, 1.0, 0.1])
print(f"logits:  {logits}")
print(f"softmax: {softmax(logits)}")
print(f"sum:     {softmax(logits).sum():.4f}")
```

### 步骤 2：缩放点积注意力

核心函数。接收 Q、K、V 矩阵，返回注意力输出和权重矩阵。

```python
def scaled_dot_product_attention(Q, K, V):
    dk = Q.shape[-1]
    scores = Q @ K.T / np.sqrt(dk)   # 缩放点积
    weights = softmax(scores)         # 转为概率分布
    output = weights @ V              # 加权求和
    return output, weights
```

### 步骤 3：带可学习投影的自注意力类

完整的自注意力模块，包含用 Xavier 式缩放初始化的 Wq、Wk、Wv 权重矩阵。

```python
class SelfAttention:
    def __init__(self, d_model, dk, dv, seed=42):
        rng = np.random.default_rng(seed)
        scale = np.sqrt(2.0 / (d_model + dk))
        self.Wq = rng.normal(0, scale, (d_model, dk))
        self.Wk = rng.normal(0, scale, (d_model, dk))
        scale_v = np.sqrt(2.0 / (d_model + dv))
        self.Wv = rng.normal(0, scale_v, (d_model, dv))
        self.dk = dk

    def forward(self, X):
        Q = X @ self.Wq
        K = X @ self.Wk
        V = X @ self.Wv
        output, weights = scaled_dot_product_attention(Q, K, V)
        return output, weights
```

### 步骤 4：在句子上运行

为一个句子创建假嵌入，观察注意力权重。

```python
sentence = ["The", "cat", "sat", "on", "the", "mat"]
n_tokens = len(sentence)
d_model = 8
dk = 4
dv = 4

rng = np.random.default_rng(42)
X = rng.normal(0, 1, (n_tokens, d_model))

attn = SelfAttention(d_model, dk, dv, seed=42)
output, weights = attn.forward(X)

print("注意力权重（每行：该令牌关注的位置）：\n")
print(f"{'':>6}", end="")
for token in sentence:
    print(f"{token:>6}", end="")
print()

for i, token in enumerate(sentence):
    print(f"{token:>6}", end="")
    for j in range(n_tokens):
        w = weights[i][j]
        print(f"{w:6.3f}", end="")
    print()
```

### 步骤 5：用 ASCII 热力图可视化注意力

将注意力权重映射为字符，快速可视化。

```python
def ascii_heatmap(weights, tokens, chars=" ░▒▓█"):
    n = len(tokens)
    print(f"\n{'':>6}", end="")
    for t in tokens:
        print(f"{t:>6}", end="")
    print()

    for i in range(n):
        print(f"{tokens[i]:>6}", end="")
        for j in range(n):
            level = int(weights[i][j] * (len(chars) - 1) / weights.max())
            level = min(level, len(chars) - 1)
            print(f"{'  ' + chars[level] + '   '}", end="")
        print()

ascii_heatmap(weights, sentence)
```

## 实战使用

PyTorch 的 `nn.MultiheadAttention` 做的正是我们构建的，再加上多头分割和输出投影：

```python
import torch
import torch.nn as nn

d_model = 8
n_heads = 2
seq_len = 6

mha = nn.MultiheadAttention(embed_dim=d_model, num_heads=n_heads, batch_first=True)

X_torch = torch.randn(1, seq_len, d_model)

output, attn_weights = mha(X_torch, X_torch, X_torch)

print(f"输入形状：            {X_torch.shape}")
print(f"输出形状：           {output.shape}")
print(f"注意力权重形状： {attn_weights.shape}")
print(f"\n注意力权重（多头平均）：")
print(attn_weights[0].detach().numpy().round(3))
```

关键区别：多头注意力并行运行多个注意力函数，每个都有自己的 Q、K、V 投影，大小为 dk = d_model / n_heads，然后拼接结果。这让模型能同时关注不同类型的关系。

## 关键术语

| 术语 | 人们怎么说 | 实际含义 |
|------|-----------|---------|
| 查询（Query, Q） | "问题向量" | 输入的可学习投影，表示该令牌在寻找什么信息 |
| 键（Key, K） | "标签向量" | 表示该令牌包含什么信息的可学习投影，用于与查询匹配 |
| 值（Value, V） | "内容向量" | 携带实际信息的可学习投影，根据注意力分数聚合 |
| 缩放点积注意力 | "注意力公式" | softmax(QK^T / sqrt(dk)) @ V——缩放防止高维下 softmax 饱和 |
| 自注意力 | "令牌看自己和别人" | Q、K、V 全部来自同一序列的注意力，让每个位置关注所有其他位置 |
| 注意力权重 | "关注程度" | 位置上的概率分布，由缩放点积上的 softmax 产生 |
| 多头注意力 | "并行注意力" | 用不同投影运行多个注意力函数，拼接结果以获取更丰富的表示 |

## 📝 教师备课总结与读后感

### 一、文档整体评价

这是一篇极为扎实的自注意力入门教材。从"数据库查找"类比切入，用纯 NumPy 按层构建完整实现，顺带覆盖数值稳定性、可视化、PyTorch 等价实现等工程要点。特别值得称赞的是"术语表"部分——每个术语同时给出"人们怎么说"和"实际含义"，这消除了初学者最大的认知障碍：概念与实现的分离。

### 二、知识结构梳理

- **基础层**：softmax 数值稳定性、点积注意力公式、三个可学习权重矩阵的作用
- **模式层**：注意力矩阵的行语义、缩放因子 sqrt(dk) 的梯度意义、因果掩码机制
- **应用层**：从自注意力到多头注意力的自然过渡、PyTorch 原生 API 的等价性、生产级实现与教学实现的差距

### 三、核心洞察

1. **自注意力 = 软性数据库查找**：传统键值查找是精确匹配，自注意力是加权混合——"谁匹配我"从二元变成连续。这是理解全部注意力变体的基石。
2. **sqrt(dk) 不是细节，是生死线**：高维点积飙高→softmax饱和→梯度消失→训练停滞。这个缩放因子是 Transformer 收敛的前提条件，不是锦上添花。
3. **每个令牌查所有令牌在一拍内完成**：RNN 需要50步才能把信息从位置1传到位置50。自注意力一步完成——这就是 Transformer 的可扩展性来源。
4. **Q/K/V 是三个不同的"眼镜"**：同一个嵌入穿上三套不同的可学习投影，看出去的世界完全不同——一个在问、一个在答、一个在传递——这就是多头注意力的前置理解。
5. **掩码不是 hack，是架构选择**：因果掩码把双向注意力变成自回归，同一个计算图，只差一个掩码矩阵——这意味着编码器-解码器共享同一个数学核心。
6. **Xavier 缩放初始化的命名暗示了它的意图**：不是随便初始化的——doc 里 scale = sqrt(2/(d_model+dk)) 是经过推导的，目的是让前向传播和反向传播的方差保持稳定。
7. **术语澄清的价值比代码本身更高**：一个学生把"query/key/value"理解为"问题/标签/内容"和理解为"三组可学习矩阵的投影结果"，是完全不同的两个层次。

### 四、教学建议

1. **先用数据库类比讲概念，再写代码**：不要一上来就抄 softmax(QK^T/sqrt(dk))V 公式。先让学生理解"查询→匹配→加权聚合"的直觉，公式自然浮现。
2. **强制学生给注意力矩阵的每一行写一句话解释**：比如"cat 这个令牌主要在关注自己（0.52）和 sat（0.14），因为它需要确定自己的语法角色"。这个练习能让抽象矩阵变成可感知的结构。
3. **对比实验：有/无 sqrt(dk) 的梯度变化**：分别跑一次带缩放和不带缩放的训练，打印梯度范数，学生会在几秒钟内理解为什么这一步不可省略。
4. **用三种掩码做对比：无掩码、因果掩码、自定义掩码**：让学生看到注意力矩阵在不同约束下的形态变化，causal mask 的那个上三角零值区域是最直观的。
5. **读 Jay Alammar 的 Illustrated Transformer 配合本课**：他的可视化与本课的代码实现是互补的——一个给直觉，一个给肌肉记忆。
6. **把 Q、K、V 的计算画成三个平行的线性层**：太多学生把这三者混为一谈。Whiteboard 上画三个独立的矩阵乘法，强调"同一个 X，三个不同投影"。
7. **练习 2（从零实现多头注意力）是必做的**：只有亲手把 Q/K/V 切成 heads 再拼回来，才能真正理解"并行注意力"不是魔法，是 `reshape + transpose`。

### 五、值得补充的内容

1. **Cross-attention 的对比**：本课只讲了 self-attention（Q/K/V 来自同一序列）。可以补充 cross-attention（Q 来自解码器，K/V 来自编码器），讲清楚 encoder-decoder 架构中两者的分工。
2. **注意力权重的梯度流分析**：反向传播时，softmax 的雅可比矩阵如何影响注意力权重的更新——这是导致注意力头 collapse 的根本原因之一。
3. **Flash Attention 的动机预告**：这个 N×N 矩阵在长序列时爆显存，引出后续的 KV Cache 和 Flash Attention 两课——本课可以作为悬念铺垫。
4. **Sparse Attention 家族概览**：讲完全注意力后，顺便提一句 Sliding Window、Longformer 等稀疏变体，让学生知道全注意力不是唯一选择。
5. **位置编码的必要性**：本课的自注意力是置换等变的，不加位置编码的话 "the cat sat" 和 "sat the cat" 完全一样——这是引入位置编码的自然动机。

### 六、一句话总结

自注意力的本质不是"每个令牌看所有令牌"，而是"每个令牌通过可学习的投影，学会以最小的信息损失有选择地聚合全局上下文"——sqrt(dk) 是让这种学习在数值上可行的保险丝。

# 🎓 Agent 架构课：自注意力——为什么并行化是唯一出路？

**副标题：从 RNN 的瓶颈到 Transformer 的暴力美学**

---

好，我想先问你一个问题——

如果你设计一个语言系统，每个词都要参考之前所有的词来做决定，你会怎么组织计算？

……

你现在脑子里可能有两种答案。一种是"一步一步来"——处理完第一个词，记住它，处理第二个，记住它们，以此类推。这是 RNN 的做法。另一种是"全部摊开一起看"——把所有词同时摆在桌面上，每个词同时问每一个其他词"你对我有多重要？"，然后一次性算出答案。这是自注意力的做法。

**我选第二种。九年来的所有生产经验告诉我，第一种是死路。**

让我解释为什么。

RNN 的核心问题是"压缩损失"。你有一个固定大小的隐藏状态向量——假设512维。你要把前50个词的语义全部塞进这512个数字里。这叫信息瓶颈。LSTM 加了门控来选哪些留下哪些丢掉，但这只是给你的压缩算法多了一点控制——它没有消除压缩的本质。到第50个词的时候，第1个词的细节已经被噪声淹没了。

自注意力的答案是：不压缩。每个词都直接看所有其他词，没有中间代理。计算量是 O(N²) 而不是 O(N)，但信息通路也是 O(N²) 而不是 O(1)。**这是我这辈子见过的最公平的交易。**

现在，让我给你两条路对比一下：

**路径一：RNN + 注意力（Bahdanau 2014）。** 你用 RNN 编码输入，然后在解码时用注意力去"回看"编码器的各个位置。好了一点。但编码器还是 RNN——它把信息压进隐藏状态那一刻已经丢了东西。注意力只能从一杯榨过的果汁里复原料，而这杯果汁就是隐藏状态。

**路径二：纯注意力（Vaswani 2017）。** 干掉 RNN。干掉卷积。只有注意力。每个位置直接投射到 Q、K、V，然后做全局加权。编码器的每一步都不压缩——它只是把信息重组，等着解码器来查。

我在生产环境里选路径二，没有犹豫过一秒。

但这就是那个"但是"了——路径二不是免费的。O(N²) 的内存和计算意味着，当你的序列长度超过一千个令牌时，那 N×N 的注意力矩阵会让你的 GPU 显存尖叫着报警。这才有了 KV Cache（第3课要讲的）和 Flash Attention 的诞生。

我给你看几个具体的数字，让你理解这个"为什么我在乎"：

**关于 sqrt(dk) 的真相。** 很多人以为除以 sqrt(dk) 是为了"数值稳定"。这只是表象。真正原因是：如果 dk=64，两个随机向量点积的方差大约是 dk 本身。不做缩放的话，softmax 的输入可能有正负几十的值——softmax 在这些区域基本是阶跃函数，梯度为零。你的反向传播死透了。除以 sqrt(dk) 把方差压回 ~1，让 softmax 处于"有梯度的区域"。**这不是 nice-to-have，这是收敛的前提。**

**关于那三个权重矩阵。** 太多工程师把 Wq、Wk、Wv 当作三个"差不多的东西"。我向你保证，它们在训练好的模型里截然不同。Wq 决定了"我在寻找什么样的语义关系"；Wk 决定了"我能给什么样的关系提供证据"；Wv 决定了"如果你选中我，我会传递什么具体信息"。拆开这三者是 Transformer 最大的架构创新——比多头还本质。因为一旦你把查询和内容分开，你就可以设计不对称的注意力：一方有权力问，另一方有义务答。

**一个反模式我必须点名批评。** 有些初学者会在 SelfAttention 的 forward 里直接 `return output` 而不返回 `weights`。我理解你为什么这么做——下游只需要 output 做计算。但这是一个巨大的调试盲区。注意力权重是你唯一的窗口，让你看到模型到底在看什么。如果某个头完全塌缩到第一个令牌上（我见过无数次），你不会从 output 里发现——你需要看 weights 的热力图才能诊断。**永远不要在生产代码中丢弃注意力权重，至少保留一个 debug flag。**

还有一个反模式：不归一化就送到 softmax。我已经见过三次不同的团队在因果掩码的实现里忘了先做 softmax mask——他们把掩码位置的分数设为零，而不是负无穷。结果是那些位置的权重是 1/n 而不是 0。这个 bug 可以存活数周，因为模型仍然会产生"看起来合理"的输出，但注意力分布是错的——性能损失集中在长序列上，debug 难度指数级。

**关于教案里提到的因果掩码练习。** 我强烈建议你做完。不是因为你需要自己实现掩码机制——PyTorch 已经帮你做好了——而是因为你需要亲自看到那块上三角变黑。那个瞬间你会理解：**同一个自注意力，加个掩码就是解码器，不加就是编码器。Transformer 之所以是统一架构，不是因为 encoder 和 decoder 用了不同的东西，而是因为它们用了同一个东西的不同配置。**

最后，给你一个清单——如果你正在设计一个基于自注意力的系统：

- ✅ 你理解 Q/K/V 三个投影各做什么吗？——不是背公式，是理解它们的语义角色。
- ✅ 你知道为什么是 sqrt(dk) 而不是除以 dk 或 sqrt(d_head) 吗？——因为方差是 dk，标准差是 sqrt(dk)，除以标准差才把分布缩放到单位方差。
- ✅ 你的注意力权重有可视化的 debug 通道吗？——至少一个热力图、一个 head-summary。
- ✅ 如果你要支持长序列（>2K），你准备好 Flash Attention 了吗？——没有的话，你的 O(N²) 显存会把单卡吃爆。

**金句：自注意力不是"让每个令牌看所有其他令牌"——是"让每个令牌用可学习的标准，有选择地重构对序列的全局理解"。那三个权重矩阵，才是核心。**

回到开头的问题——为什么并行化是唯一出路？不是因为 RNN 的串行不够快。而是因为**任何串行架构都有一个不可逃避的信息压缩步骤**。自注意力的暴力美学在于：它拒绝压缩，它把信息保存在所有位置，让每个决策都能引用到原文——而不是引用到原文的一个摘要。

这就是为什么全世界的搜索、推荐、对话、编码系统都在用 Transformer。不是因为它"新"。是因为它"不丢信息"。
