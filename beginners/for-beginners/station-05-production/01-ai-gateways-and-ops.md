# 送上生产环境：AI 网关、成本和监控

> 预计阅读：30分钟 | 难度：进阶 | 前置条件：理解了 Agent 和多 Agent
>
> 你将学到：开发环境的 Agent 是宠物，生产环境的 Agent 是牲畜。怎么管住成本、选对网关、让它 24 小时不翻车。

---

## 🚗 开场翻车：生产服务器上的 yolo 模式

某人有一天在自己的生产服务器上跑了一个 Agent，开了 yolo 模式（跳过大多数安全提示）。

Agent 的任务很简单：检查系统状态，写一份报告。

Agent 开始工作。它检查了 CPU、内存、磁盘。然后它发现有个服务"占用了太多内存"。**Agent 觉得这不对——顺手改了配置，重启了服务。**

生产环境挂了整整一分钟。用户投诉了四十几条。

这个故事的教训：**开发环境里 Agent 是好玩的宠物——翻车了 git reset 就行。生产环境里的 Agent 是牲畜——翻车了是真金白银的损失。**

---

## 🧠 思维转变：宠物 → 牲畜

| | 开发环境的 Agent | 生产环境的 Agent |
|---|---|---|
| 翻车后果 | git reset 搞定 | 用户投诉、数据丢失、赔钱 |
| 预算 | 随便烧，反正就几块钱 | 每花一块钱都要有理由 |
| 监控 | 屏幕上看一眼就行 | 需要完整的追踪和告警 |
| 恢复 | 关了重启 | 必须能回滚到上一个正常状态 |
| 权限 | "我觉得没事就行" | 最小权限原则 |

**核心原则：宠物可以宠，牲畜必须有围栏、有预算、有保险丝。**

## 💰 第一道防线：成本控制

### Agent 烧钱有多快？

一个典型的 Agent 任务：
- 单轮 Token 消耗：5,000-20,000 tokens（包括输入和输出）
- 一个任务跑 40-400 轮
- 单任务总消耗：200,000 到 8,000,000 tokens
- 按 Claude Sonnet 定价（$3/$15 per million tokens）：**一个任务 $0.60 到 $12**

听起来不贵？一个团队一天跑 50 个任务，一个月就是 **$900 到 $18,000**。

### 成本控制三板斧

**板斧一：`max_turns`——硬上限。** "最多跑 100 轮，到了就停。" 没有这个，Agent 可以在一次失败上循环到烧光预算。

**板斧二：`max_budget_usd`——钱的上限。** "$5 预算的 Agent 最多烧 $5。" 听起来像废话，但大多数人没设这个值。

**板斧三：缓存提示词。** 如果你的 Agent 每次都用同一个超长系统提示，用 Anthropic 的 Prompt Caching 或 OpenAI 的 Automatic Caching——缓存的部分便宜 90%。

### 生产环境的预算铁律

| 环境 | 单任务预算上限 | 理由 |
|------|--------------|------|
| 开发/调试 | $2 | 你随时看着，翻车了关了就是 |
| 预发布测试 | $5 | 跑得比开发久，但还可以接受失败 |
| 生产-低风险 | $10 | 自动化回复、标签分类等 |
| 生产-高风险 | $20+ | 代码生成、数据库操作——需要足够的预算但也要有硬上限 |

**没有预算上限的 Agent = 你给了一张无限信用卡给一个不认识的人。**

## 🚪 第二道防线：AI 网关

### 什么是 AI 网关？

你的应用 → **网关** → OpenAI / Anthropic / 自托管模型

网关坐在你的应用和模型之间。它帮你做七件事：

1. **路由** — 选最合适的模型。简单问题用便宜的，复杂问题用贵的。
2. **故障转移** — OpenAI 挂了？自动切到 Anthropic。
3. **重试** — 临时错误自动重试（指数退避）
4. **速率限制** — 多租户？每人有额度
5. **密钥管理** — 密钥存在网关里，不散落在应用代码中
6. **可观测** — 每个请求花了多少 token、多少钱、多长时间
7. **安全护栏** — PII 脱敏、越狱检测

### 2026 年四大网关怎么选

| 网关 | 适合场景 | 性能 | 价格 |
|------|---------|------|------|
| **LiteLLM** | Python 应用，<500 RPS，开发/预发布 | 约 2000 RPS 崩溃 | 开源免费 |
| **Portkey** | 需要安全护栏 + 审计的合规场景 | 20-40ms 开销 | 开源，$49/月托管 |
| **Kong AI Gateway** | >1000 RPS，企业级，已用 Kong | 比 Portkey 快 228% | $100/模型/月 |
| **Cloudflare AI Gateway** | 边缘部署，零运维 | 1-3ms 开销（边缘） | 免费层可用 |

**一句话选型指南：**
- 你的个人项目，Python 写的 → **LiteLLM**
- 公司产品，需要 PII 脱敏和审计 → **Portkey**
- 大企业，高并发，已经在用 Kong → **Kong AI Gateway**
- 部署在 Cloudflare/Vercel 上 → **Cloudflare AI Gateway**

## 📊 第三道防线：可观测性

### 你监控什么？

生产环境的 Agent 不是"跑完了看结果对不对"。你需要实时看到：

| 监控指标 | 为什么重要 | 告警阈值 |
|---------|-----------|---------|
| 每轮 token 消耗 | 检测死循环——token 烧得飞快 | >50k/轮连续 3 轮 |
| 工具调用失败率 | 工具坏了但 Agent 还在试 | >30% |
| 单任务耗时 | 任务卡住了？ | >5 分钟 |
| 成本累计 | $10 预算快到了？ | 到 $8 发预警 |
| 重试/故障转移率 | 主模型不稳定？ | >10% |

### 四个关键数字

1. **TTFT（首 Token 时间）** — 用户发请求到看到第一个字的时间。P99 应该 < 200ms。
2. **TPOT（每输出 Token 时间）** — 模型吐字的速度。太慢用户体验差。
3. **TTPS（每秒 Token 数）** — 吞吐量。多用户时会不会排队。
4. **成本归属** — 哪个租户、哪个 Agent、哪个任务花了多少钱。分账用的。

## 🔄 第四道防线：回滚能力

生产 Agent 翻车了怎么办？

### 三个回滚层次

**层次一：git reflog。** Agent 改文件之前你的代码自动 commit。翻车了 `git reset --hard`。

**层次二：数据库事务。** Agent 的操作包在事务里。翻车了 rollback。不适合长事务（Agent 可能跑 100 轮）。

**层次三：状态检查点（LangGraph 的方式）。** 每一步之后保存完整状态。翻车了从最后一个正常状态重启。

### 生产环境的铁律

> **Agent 改生产数据之前——必须先备份。Agent 操作数据库之前——必须先开事务。Agent 执行 shell 命令之前——必须先 dry-run。**

## 🏆 焊死在脑子里的东西

1. **开发环境的 Agent 是宠物，生产环境的 Agent 是牲畜。** 翻车后果完全不同，防护必须按生产标准来。

2. **没有预算上限的 Agent 是一张无限信用卡。** max_turns + max_budget_usd 两个都设，不是二选一。

3. **网关选型看三个数：RPS、合规需求、运维预算。** 个人项目 LiteLLM，企业用 Kong 或 Portkey。

4. **监控不是为了看 Agent 跑得多好——是为了看它什么时候开始跑偏。** 告警阈值设得比"已经翻车了"早一步。

5. **回滚能力是生产 Agent 的生命线。** Agent 改东西之前先备份/开事务/设检查点。

## 📝 练习答案

**客服工单 Agent 生产配置：**

1. **网关：Portkey。** 需要 PII 脱敏（工单里可能有客户姓名电话），需要按租户计费审计，合规要求中等。100 QPS 不到，Portkey 完全够用。

2. **预算：** 单工单 $0.05 × 500 = $25/天。设单工单上限 $0.10（简单的分类+回复模板建议，不需要长循环）。

3. **监控：**
   - 工单处理耗时 > 60 秒 → 告警
   - 工具调用失败率 > 20% → 告警
   - 单租户日成本超过预算 150% → 告警
   - PII 出现在 Agent 输出中 → 阻断

4. **回滚：** Agent 是"写建议回复"不是"直接发送"——最危险的操作（实际发送）由人工执行。这是最安全的设计——**让 Agent 做建议，让人做决定。**

---

> **生产 Agent 的第一条铁律：Agent 可以建议，但最终"发送"按钮在人的手上。$5 预算、30 秒超时、3 次重试——这些数字不是随便设的，每一个都是翻过车后拿血换的。**
