# 构建完整 LLM 流程

> **原文：** Building a Complete LLM Pipeline  
> **预计时间：** ~20 分钟  

---

## 🧠 十二个阶段

```
分词 → 预训练 → 扩展 → SFT → RLHF/DPO → 宪法AI → 评测 → 量化 → 推理服务
```

每一个阶段消费上一个阶段的产出物，产出给下一个阶段。

---

## 🔧 核心理念：流程卫生

Llama 3 405B 消耗了约 3000 万 H100 小时。DeepSeek-V3 用了约 280 万 H800 小时。一个损坏的检查点 = 一周白干。

**四个组件：**
1. **清单** — 每个阶段的配置、种子、数据哈希
2. **编排器** — 按序运行，验证输入输出
3. **评测门禁** — 不达标的产出物不传给下一阶段
4. **产出物存储** — 带哈希的检查点

**这个模式从 1 亿参数到 1 万亿参数不变。** 区别只是数字大小，不是流程形态。

---

## 🏆 焊死在脑子里的东西

1. **LLM 训练是流水线，不是笔记本。** 12 个阶段，每个有确定性输入输出。
2. **哈希所有产出物。** 损坏的检查点会传播到下游。
3. **评测门禁 = 质量保险。** 不达标就别传入下一阶段。

---

> **训练一个 LLM 不是跑一个脚本——是管理一条流水线。一个损坏的检查点会污染八个下游阶段。流程卫生不是洁癖，是生存技能。**
