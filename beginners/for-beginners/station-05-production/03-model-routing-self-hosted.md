# 省钱的秘诀：模型路由 + 自托管引擎选型

> 预计阅读：25分钟 | 难度：进阶 | 前置条件：理解了 AI 网关和推理基础
>
> 你将学到：简单问题用便宜模型、复杂问题用贵模型——一年能省六位数。以及如果要自己托管模型，五大引擎怎么选。

---

## 🚗 开场翻车：所有问题都用 GPT-5，一个月烧了 $80,000

一个 SaaS 团队把所有用户请求都甩给了 GPT-5（最贵的模型）。

分析后发现：70% 的请求是"巴黎天气怎么样"、"帮我把这句话改通顺"——一个 1/30 成本的便宜模型就够了。

**他们每月在不需要推理能力的问题上烧了 $56,000。**

解决方案叫**模型路由**：简单问题 → 便宜模型。复杂问题 → 贵模型。同样的质量，账单打 3-4 折。

---

## 🔀 模型路由：怎么判断"该用哪个模型"

### 四种判断信号

| 信号 | 怎么用 | 例子 |
|------|--------|------|
| **任务分类** | 先判断任务类型，再选模型 | "聊天" → 便宜，"写代码/数学" → 贵 |
| **提示长度** | 长提示 → 贵模型 | >4000 token → 需要强推理保持连贯 |
| **嵌入相似度** | 跟已知困难问题像 → 贵模型 | "解释量子计算" 跟困难集里的问题相似 |
| **首轮置信度** | 先让便宜模型答，不确定就升级 | 便宜模型回答含"我不确定" → 切贵模型 |

### 三种路由模式

| 模式 | 怎么工作 | 延迟 | 质量 |
|------|---------|------|------|
| **预路由** | 前端分类器先判断 | 最快（+5-10ms） | 好 |
| **级联** | 便宜先试，不确定切贵的 | 中（贵模型约 2x） | 最好 |
| **集成路由** | 并行跑多个，选最佳 | 最慢 | 最高 |

**实际节省：** 生产案例报告 20-60% 成本降低。一个高流量 SaaS 30% 路由效率 = **六位数年省**。

## 🖥️ 如果要自己托管模型

2026 年五大推理引擎：

| 引擎 | 最适合 | 一句话 |
|------|--------|--------|
| **llama.cpp** | CPU / 边缘 / 最少依赖 | CPU 上最快，完全自己控制 |
| **Ollama** | 笔记本开发 | 一条命令跑起来，但生产吞吐量差 3 倍 |
| **TGI** | ⚠️ 维护模式 | 2025年12月进入维护模式，新项目别用 |
| **vLLM** | 通用生产默认 | 最广的生产基线，100+ 用户首选 |
| **SGLang** | Agent 多轮场景 | 400,000+ GPU 在生产，前缀缓存专家 |

### 选型决策树

```
什么硬件？
  ├── 只有 CPU → llama.cpp（没别的选择）
  ├── AMD GPU → vLLM
  ├── NVIDIA Hopper → vLLM 或 SGLang
  └── NVIDIA Blackwell → TRT-LLM 或 vLLM

多少用户？
  ├── 1 个（本地开发）→ Ollama
  ├── 10-100 → vLLM 单 GPU
  └── 100+ → vLLM 生产栈 或 SGLang

什么负载？
  ├── 通用聊天 → vLLM
  ├── Agent 多轮 → SGLang（RadixAttention 缓存优势）
  └── 长上下文 128K+ → vLLM + 分块预填充
```

### 标准流水线

```
开发（Ollama）→ 预发布（llama.cpp）→ 生产（vLLM/SGLang）
                 相同的模型权重（GGUF/HF）
```

## 🎮 互动练习

一个团队：每天 10,000 个 API 请求，70% 简单聊天，30% 代码生成。当前全用 GPT-5，月账单 ~$15,000。设计路由方案。

---

## 🏆 焊死在脑子里的东西

1. **不是所有问题都值得用最贵的模型。** 70% 的请求用便宜模型就够了。

2. **级联路由质量最好。** 便宜先试 → 不确定切贵的 → 约 10% 流量多等 2x 延迟，省 50%+ 成本。

3. **路由漂移是真实风险。** 用户行为会变。用在线质量指标（升级率、用户踩赞比）持续监控。

4. **自托管选引擎看三样：硬件、规模、负载。** CPU 只能用 llama.cpp。Agent 多轮场景用 SGLang。通用选 vLLM。

---

## 📝 练习答案

**路由方案：**
1. 预路由：70% 简单聊天 → Haiku 级别（约 $0.25/M tokens）
2. 30% 代码生成 → GPT-5（约 $3-10/M tokens）
3. 级联兜底：便宜模型置信度低的 → 升级到 GPT-5

**预期效果：** 月账单从 $15,000 → 约 $4,000-6,000（省 60-70%）

**监控：** 升级率 >30% → 便宜模型过载了。用户踩赞比下降 → 质量在漂移。
