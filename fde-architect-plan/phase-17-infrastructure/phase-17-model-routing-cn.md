# 模型路由作为成本削减基元

> 一个动态代理评估每个请求（任务类型、Token 长度、嵌入相似度、置信度），将简单查询发送到便宜模型，将复杂查询升级到前沿模型。也称为模型级联（model cascading）。生产案例研究显示，在英国/美国/欧盟部署中，同质量水平下可降低 20-60% 的成本；高流量 SaaS 上 30% 的路由效率提升转化为六位数的年节省。2026 年的背景是 LLM 推理价格每年下降约 10 倍 — GPT-4 级别的 Token 从 2022 年底的 $20/M 降到 2026 年的约 $0.40/M。大部分降幅来自更好的服务栈（Phase 17 · 04-09），而不是硬件。路由是你将这一价格下降转化为利润的方式，同时避免产品倒退。失败模式是便宜模型漂移：路由将 40% 推给较弱模型，推理任务质量下降 3-5%，一个季度没人注意到。用在线质量指标来门禁路由，而不仅仅依赖离线评估集。

**类型：** 学习
**语言：** Python（标准库，简单的级联路由模拟器）
**前置课程：** Phase 17 · 01（托管 LLM 平台）、Phase 17 · 19（AI 网关）
**时间：** 约 60 分钟

## 学习目标

- 解释模型级联：便宜优先加置信度检查，低置信度时升级。
- 枚举四种路由信号（任务分类、提示长度、与已知困难集的嵌入相似度、首轮自置信度）。
- 计算目标路由比例和质量损失容忍度下的预期混合成本。
- 指出捕获便宜模型漂移的漂移监控指标（在线质量门禁）。

## 问题

你的服务在 GPT-5 上每月花费 $80k。分析显示 70% 的查询很简单："巴黎现在几点？""重新表述这句话。"一个 Haiku 级别的模型以 3% 的成本完美处理这些。30% 需要 GPT-5 的推理能力 — 编码、数学、多步骤规划。

如果你将 70% 路由到便宜模型，30% 路由到昂贵模型，你的账单在相同产品质量下下降约 65%。这就是路由。诀窍是构建代理而不回归质量。

## 概念

### 四种路由信号

1. **任务分类**：简单/复杂/代码生成/数学/聊天。可以是一个基于规则的分类器、一个小型 LLM（Haiku 级别，$0.25/M），或对标记桶的嵌入相似度。输出：路由 = 便宜 / 平衡 / 前沿。

2. **提示长度**：> 4K Token 的提示通常需要前沿模型以保证连贯性。< 500 Token 的提示通常不需要。

3. **与已知困难集的嵌入相似度**：如果查询接近（余弦 > 0.88）一个已知困难的桶，直接升级到前沿模型。

4. **首轮自置信度**：发送到便宜模型；如果模型的 log-probs 显示低置信度，或它拒绝回答，或输出模糊语言，则在前沿模型上重试。对约 10% 的流量增加 P95 延迟，但为其他 90% 节省 50%+。

### 三种模式

**预路由**（前端分类器）：增加约 5-10 毫秒延迟；总体最快。

**级联**（便宜优先，低置信度升级）：约 1.2 倍中位延迟（便宜运行加验证），升级请求约 2 倍。质量底线最好。

**集成路由**（对样本并行运行便宜和前沿，奖励模型选择）：质量最高、成本最高；仅用于关键 A/B。

### 实现

AI 网关（Phase 17 · 19）暴露路由功能。LiteLLM 有带回退和成本路由的 `router` 配置。Portkey 有守卫 + 路由。Kong AI Gateway 有基于插件的路由。OpenRouter 的模型市场暴露推荐 API。

开源：RouteLLM（LMSYS）、Not Diamond（商业）、Prompt Mule。

### 2026 年价格曲线

| 模型级别 | 2022 年底 | 2026 年 | 变化 |
|---------|----------|--------|------|
| GPT-4 级别质量 | ~$20/M | ~$0.40/M | 便宜 50 倍 |
| 前沿（GPT-5、Claude 4） | — | ~$3-10/M | 新层级 |

大部分改进来自服务效率 — Phase 17 · 04-09 中的核心课程转化为供应商侧的成本下降。路由让你在应用层捕获这些收益，而不是等所有用户迁移到便宜层级。

### 漂移是真正的风险

你的路由将 40% 发送到便宜模型。六个月后，任务分布发生了变化（用户变得更老练，提问更长）。路由器没有注意到，因为它的分类器是用 Q1 数据训练的。质量悄然下降。没有人足够大声地抱怨。你在竞品基准测试中才发现自己落后了。

用在线质量指标门禁路由：

- 用户对每条路由的赞/踩。
- 对每个路由的留出样本（5%）进行自动化 LLM 评判。
- 升级率：如果级联向上路由超过 30%，便宜模型正在被过度路由。
- 每条路由的拒绝率。

### 你应该记住的数字

- 2026 年同质量水平路由节省：案例研究显示 20-60%。
- 2022-2026 年 LLM 价格下降：总计约每年 10 倍。
- GPT-4 级别 2022 vs 2026：约 $20/M → 约 $0.40/M。
- 级联延迟影响：约 1.2 倍中位延迟，升级请求约 2 倍（约 10% 流量）。

## 📝 教师备课总结与读后感

### 一、文档整体评价

这是一篇"工程经济学"风格的教学文档 — 核心不是实现细节，而是成本模型和质量权衡。文档结构紧凑：先展示 $80k/月的账单带来痛感，再提出 65% 的节省空间，然后讲四种路由信号和三种路由模式，最后落到漂移监控。2022-2026 年价格曲线的 50 倍降幅数据非常有冲击力，解释了路由的经济学基础。亮点是"漂移是真正的风险"这节 — 这是两个季度后才暴露的隐蔽问题，但文档把它提升到了和路由本身同样重要的位置。不足在于没有给出具体分类器的准确率数据（任务分类本身也会出错），以及没有讨论组合路由（多信号加权）的实现。

### 二、知识结构梳理

- **经济学层**：$80k → $28k 的 65% 节省空间，2022-2026 年 50 倍价格降幅，路由是将价格下降转化为利润的机制。
- **信号层**：四种路由信号（任务分类、提示长度、嵌入相似度、自置信度），各自适合的场景和成本。
- **模式层**：预路由（5-10ms 延迟）、级联（1.2x 中位延迟，质量底线最好）、集成路由（最高质量/最高成本）。三种模式的延迟-成本-质量三角。
- **风险层**：漂移风险，在线质量门禁（点赞/踩、LLM 评判、升级率、拒绝率），"静默质量下降"的隐蔽性。

### 三、核心洞察

1. **路由的本质是把"用户不付钱的复杂度"外包给便宜模型。** 70% 的查询不需要 GPT-5 的推理能力，这是前提 — 不是所有查询需要同等级别的智能。
2. **级联是质量底线最好的路由模式。** 先跑便宜模型再验证，虽然在约 10% 请求上增加延迟，但保证了最终质量不低于单独使用前沿模型。适合质量敏感的场景。
3. **漂移是最危险的失败模式，因为它静默发生。** 便宜模型渗透不是崩溃，是质量缓慢下降 3-5%。没有在线质量门禁，一个季度后才发现 — 用户已经流失了。
4. **四种信号不应独立使用，应该组合加权。** 任务分类 + 提示长度 + 嵌入相似度作为路由决策的基础，自置信度作为兜底验证。单信号路由脆弱。
5. **LLM 价格每年 10 倍降幅意味着路由策略需要不断重新校准。** 今天的"便宜模型"可能明年就足够强了，路由比例和历史数据不能一成不变。
6. **升级率是最简单也最有用的漂移监控指标。** 如果级联向上路由从 8% 升到 22%，一定有问题 — 要么任务分布变了，要么便宜模型退化了。
7. **路由不是一劳永逸的。** 至少每季度重新评估分类器准确率、升级率和每条路由的用户满意度。把它当作一个持续运营功能，不是一个一次性的架构决策。

### 四、教学建议

1. **用"你的 GPT-5 账单是多少？"作为开场问题。** 大多数学员会有一个心痛的数字。然后展示 65% 的节省空间 — 这直接转化为学习动力。
2. **价格曲线图值得单独做一页幻灯片。** 2022 $20/M → 2026 $0.40/M 的下降曲线，标注技术节点（FlashAttention、量化、vLLM、SGLang）。学员需要理解"为什么价格在降"才能正确使用路由。
3. **三种路由模式的比较可以用一个三角图。** 三个顶点是：延迟、成本、质量。预路由偏延迟-成本、级联偏质量、集成偏质量但最贵。视觉化比文字更直观。
4. **漂移的讲解用动画图表。** 横轴是时间（6 个月），两条线：一条是升级率（平稳上升），一条是用户满意度（平稳下降但无人察觉）。到第 5 个月两条线交汇时才发现问题。
5. **"什么时候路由省最多"可以用一个练习题。** 给 3 个不同流量分布（80% 简单 / 50% 简单 / 20% 简单），让学员计算级联的节省率。理解"简单查询占比越高，路由越有价值"。
6. **升级率的阈值讨论。** 让学员分组辩论"升级率到什么程度应该触发什么行动？"8% → 正常，15% → 调查，22% → 重构分类器，30% → 停用便宜路由。没有标准答案，培养判断力。
7. **强调路由不是"省钱了就可以忘了"。** 路由系统需要至少三个仪表板：成本每路径、质量每路径（LLM 评判）、升级率。没有监控的路由是盲飞。

### 五、值得补充的内容

1. **多模型提供商的路由策略。** 不是选两个 OpenAI 模型，而是 OpenAI 便宜 + Anthropic 昂贵，甚至混合 Google Gemini。不同提供商的模型各有优劣，路由策略更复杂。
2. **分类器本身的成本和延迟。** Haiku 级分类器每次分类约 $0.00003，但如果每天 100 万请求，分类器自己一个月就是 $900。分类器的成本是否值得？需要数学。
3. **A/B 测试框架对路由的意义。** 如何安全地测试一条新路由规则而不影响真实用户体验。分流量、指标监测、回滚策略。
4. **路由 + 缓存的叠加效应。** 便宜模型路由的前缀缓存命中可以进一步降低成本。路由和缓存不是互斥的，它们可以堆叠。
5. **边缘案例：当便宜模型输出看似正确但实际错误时。** 自置信度检查不总是能捕获这类错误 — 模型自信地把错误当正确。需要额外的验证层。

### 六、一句话总结

**模型路由是 LLM 经济学中最优雅的成本削减手段：70% 的简单查询不值得花 97% 的预算；关键不是路由的技术实现，而是用在线质量门禁防止便宜模型静默漂移。**

# 🎓 Agent 架构课：模型路由——为什么你的 GPT-5 账单里有一堆"巴黎现在几点"

同学们好。我是你们的FDE工程老师，今天讲的是 模型路由。

**你上个月花了 $74,000 在 AI API 上。翻日志发现 40% 的请求是"巴黎现在几点"——一个 Haiku 级别模型就够了的查询，你用了 GPT-5。**

模型路由的核心思想：不是所有请求都需要最强模型。简单查询（FAQ、天气、翻译）走低成本模型。复杂推理（代码生成、数学证明、长文档分析）走强模型。

路由策略：① **分类器路由**——一个小模型先判断复杂度，决定走大模型还是小模型。② **级联路由**——先尝试小模型，如果置信度不够再升级到大模型。③ **语义路由**——根据查询嵌入选择最匹配的模型。

## 结课清单

1. **不是所有请求都需要最强模型。** 简单查询 = 便宜模型。
2. **分类器→级联→语义，三种路由策略递进。** 从简单规则到语义匹配。
3. **一个分类器的成本 << 40% 请求走错模型。** ROI 立竿见影。

最后一句话：**你的 AI 账单上最大的浪费不是"模型太贵"——是"简单的请求走了贵的模型"。加一个路由器，省 40%。**

# 💼 从业者故事：模型路由——为什么你的 GPT-5 账单里有一堆"巴黎现在几点"

年初财务发了一份 AI API 消费明细给我，我对着屏幕愣了两分钟。上个月花了 $74,000。然后我做了一件每个工程师都该做的事：翻日志，看这些钱花在哪了。

结果让人崩溃。70% 的请求是这些："总结一下这段文字"、"把这句话翻译成英文"、"这篇文档的重点是什么"。全是 GPT-4 级别甚至更便宜的模型就能完美处理的任务。但我们全扔给了 GPT-5。

这就好比你用一辆法拉利去买菜。法拉利确实能买菜，但五菱宏光也能买，而且便宜 30 倍。你每天开法拉利去菜市场，不是因为需要法拉利，是因为你只有法拉利。

模型路由解决的问题就是这个：**给你一辆五菱宏光（便宜模型）处理日常通勤，法拉利（前沿模型）只在需要飙车的时候出动。** 算一笔简单的账：70% 的流量花 3% 的成本，30% 的流量花 97% 的成本。混合成本约 $0.40 × 30% + $0.04 × 70% = $0.15 per unit，比全用 GPT-5 便宜 60%+。

## 四种路由信号：别只用一种

我最开始只用了提示长度一个信号——超过 4K Token 的就升级到 GPT-5。看起来逻辑自洽——长提示通常需要更强的推理。但实际上很多长提示是"把这篇 5000 字的文档总结成一段话"，这个任务 Haiku 级别的模型做得飞起。分类错了。

后来我把四种信号全部用上了——任务分类、提示长度、与已知困难集的嵌入相似度、首轮自置信度——效果好了很多。不是每一种都要用，但是只用一种就像闭一只眼看路，迟早撞墙。

**前置路由**最快：一个轻量级分类器在 5-10ms 内判断往哪走。但分类器自己也是模型，也会出错。有些看起来简单的查询其实是陷阱——"请解释量子纠缠"看起来短，但其实需要一个解释得很好的模型。

**级联路由**最稳：先送给便宜模型跑一遍，然后检查置信度。置信度低或输出模糊就重新用前沿模型跑。代价是约 10% 的请求多等一倍的时间，换来的是质量不下降。质量敏感场景（代码、医学、法律）必须用级联，没有商量。

**集成路由**最贵但最好：同时跑两个模型，让第三个人（奖励模型）选最佳答案。只适用于 A/B 实验和关键决策场景。你不可能把 100% 流量都用集成路由，那是月耗百万的玩法。

## 漂移：你省了钱，丢了质量，两个月后才发觉

这是一个真实的事故。我们在 Q2 把 40% 的流量路由到了便宜模型，一切正常。Q3 结束的时候发现用户满意度缓慢下跌了 3%。排查了两周没找到原因。最后发现任务分布变了——用户从"简单翻译"慢慢进化到了"多步骤推理"。路由器的分类器是 Q1 数据训练的，它不知道用户在成长。

便宜模型静默漂移是最危险的失败模式。它不会报警，不会崩溃，不会产生 5xx 错误。它只是让你的产品质量缓慢而下滑，直到竞品的 benchmark 超过你，你才恍然大悟。

防御手段就一个：**在线质量门禁。** 每一条路由路径采样 5% 的流量做 LLM 自动评判，跟踪点赞/踩比例、升级率（级联路由向上重跑的占比）、拒绝率。我给团队设定的警戒线：升级率超过 30% → 便宜模型不合适，降低路由比例；满意度下降 2% 且持续一周 → 冻结路由规则，回到全量前沿模型直到找到原因。

数字是冷冰冰的谈判专家——"我觉得便宜模型还行"没用，升级率从 8% 涨到 22% 就是你的证据。

## 价格曲线的背后：为什么 2022 到 2026 便宜了 50 倍

2022 年底，GPT-4 级别的 API 价格是 $20/M Token。2026 年，约 $0.40/M Token。降了 50 倍。这个速度比摩尔定律快得多。

降价的真正驱动力大部分不是新硬件，是推理优化技术——FlashAttention（注意力计算加速）、KV 缓存量化和复用、vLLM 的 PagedAttention、SGLang 的 RadixAttention。这些技术我们在这个课程的第 4-9 课已经全部覆盖了。

然后你再叠加上模型路由——便宜模型处理简单任务占 70%，前沿模型处理复杂任务占 30%——总体 Inference 成本可以再砍 60%+。你不是在等供应商降价，你是在把"供应商已经降下来的价格"通过路由策略映射到你的真实流量上。

这个逻辑不是"抠门"，是工程经济学。如果你每年在 LLM 推理上花 100 万美元，路由策略可以把这笔钱降到 30 万以下。省下的 70 万可以多招两个工程师，可以做更长的上下文实验，可以把省下来的算力用在更复杂的能力上而不是"巴黎现在几点"上。

**模型路由不是一个省钱的小技巧，它是 LLM 经济学的必修课。如果你的流量分布是纺锤形的（大部分简单、少量复杂），而你还没做路由，你每个月都在用法拉利买白菜。这不是技术能力问题，是财务纪律问题。**
