# 分词器：你的 LLM 不认识字——只认识数字

> **原文：** Tokenizers: BPE, WordPiece, SentencePiece  
> **预计时间：** ~40 分钟  

---

## 🚗 开场翻车：同样的意思，不同的价格

你问 Claude "帮我写段代码"。它回了一个精美函数。账单：$0.03。

你用中文问"请帮我编写一段计算机程序代码"。同样的意思。但中文每个字可能是一个 token，英文一个词可能是一个 token。**同样的意思，不同的字数，不同的价格。**

更重要的是："unfortunately"（一个词）在某些分词器里被拆成 `["un", "fortun", "ately"]`三个 token。本来一个词能表达的意思，占了三个槽位。你的 128K 上下文窗口实际能装的意思缩水了 75%。

**分词器不是预处理的细节——是架构。你用哪个分词器，决定了你的模型看世界的方式。**

---

## 🧠 三种切法，一种活法

### 按词切 → 词汇表爆炸

```
"The cat sat" → ["The", "cat", "sat"]
```

简单。但"tokenization"这个词呢？"GPT-4o"呢？德语复合词"Geschwindigkeitsbegrenzung"呢？

英语有超过 100 万种词形。加上代码、URL、100 种语言——你需要一个无限大的词汇表。不在词汇表里的词 → `[UNK]`（模型说"我不认识"）。

### 按字符切 → 序列太长

```
"hello" → ["h", "e", "l", "l", "o"]
```

永远不会出现未知字符。但 10 个词的句子变成 50 个字符。模型要在"t"+"h"+"e"= "the"这种三岁小孩都知道的事上浪费注意力。

### 按子词切 ✅ → 黄金分割

```
"unhappiness" → ["un", "happi", "ness"]
```

常用词保持完整，罕见词分解成有意义的片段。词汇表可控（30K-128K），序列短，基本没有 `[UNK]`。

**每个现代 LLM 都用子词分词。** GPT-4、Claude、Llama、Gemini——全部。

---

## 🔧 BPE：字节对编码

BPE 是一种贪心压缩算法，被借用来做分词。核心思想：

```
1. 从单个字符开始
2. 统计所有相邻字符对的频率
3. 合并频率最高的那一对 → 变成新 token
4. 重复，直到达到目标词汇量
```

### 一个小例子

```
语料: "lower"×5, "lowest"×2, "newest"×6

初始: l o w e r, l o w e s t, n e w e s t

第1轮: 统计相邻对 → (w,e) 出现 13 次 → 合并为 "we"
结果: l o we r, l o we s t, n e we s t

第2轮: 统计 → (we,s) 出现 8 次 → 合并为 "wes"
结果: l o wes r, l o wes t, n e wes t

...继续直到词汇表满了
```

### BPE 的局限

- 不知道"意义"。只知道频率。`"est"`在 `"largest"` 里是有意义的后缀，但在 `"restaurant"` 里不是——BPE 不管这个。
- 跨语言坑。英语训练的分词器切中文很烂——一个汉字可能被切成多个 token。

---

## 📊 三种分词算法

| 算法 | 策略 | 谁在用 |
|------|------|--------|
| **BPE** | 从字符开始，合并最频繁的对 | GPT 系列（tiktoken） |
| **WordPiece** | 合并最大化训练数据似然的 pair | BERT |
| **Unigram** | 从大词汇表开始，逐步删减 | SentencePiece（Llama、Gemini） |

## 🏆 焊死在脑子里的东西

1. **分词器 = 模型看世界的语言。** 换分词器 = 换模型对文本的理解方式。
2. **子词分词 = 黄金分割。** 常用词不拆，罕见词拆成有意义的片段。
3. **BPE 看频率，不看意义。** `"est"`在`"largest"`和`"restaurant"`里被同等待遇。
4. **中文和代码被分词器坑得最惨。** 多音节语言和编程语言的分词效率远低于英语。

---

## 📝 练习答案

```
输入 "abc" → tokenize → ["ab", "c"]
BPE 从最长匹配开始：先匹配 "ab"（在词汇表里），剩下 "c"（也在词汇表里）
```
