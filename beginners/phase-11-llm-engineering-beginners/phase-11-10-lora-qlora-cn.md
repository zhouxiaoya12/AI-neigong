## 🧠 LoRA 核心洞察

微调时权重矩阵的变化是**低秩**的。一个 4096×4096 的矩阵有 1677 万个参数——但有用的变化可以用两个小矩阵相乘来表示：

```
正常: y = Wx          (W 是 4096×4096 = 16,777,216 参数)
LoRA: y = Wx + BAx    (B 是 4096×16, A 是 16×4096 = 131,072 参数)
                                              ↓
                                    只训练 0.78% 的参数!
```

**A 随机初始化，B 初始化为零。** 一开始 LoRA 贡献为零，模型保持原始行为。训练过程中逐渐学到适配。

```
输入 x ──→ 冻结的 W (原始权重，不动) ──┐
输入 x ──→ A (16×d) → B (d×16) ────────┼──→ 输出 y
                                        │
                                 相加(合并)
```

## 🔧 QLoRA：再省 4 倍显存

**LoRA 训练的是 16 位精度的适配器。QLoRA 把基础模型量化到 4 位（NF4 格式），只训适配器。**

```
正常: 基础模型 fp16 (16GB) + 适配器 fp16 (50MB) = ~16GB
QLoRA: 基础模型 4bit (4GB) + 适配器 fp16 (50MB) = ~4GB
```

**Llama 3 8B 在笔记本上的 6GB 显卡就能跑。**

### QLoRA 关键技巧

```python
# QLoRA 的三大发明
# 1. NF4 量化——专门为正态分布权重设计的 4 位格式
# 2. 双重量化——连量化常数额外也量化，再省 0.4GB
# 3. 分页优化器——显存不够时把优化器状态换出到 CPU 内存
```

## 🏆 焊死在脑子里的东西

1. **LoRA = 冻结原模型 + 训两个小矩阵。** 训不到 1% 参数，质量匹配全量微调。
2. **QLoRA = LoRA + 4bit 量化。** 笔记本显卡也能跑 8B 模型。
3. **微调改变模型风格，RAG 提供外部知识。** 两者互补，不是互斥。
4. **用 Unsloth 或 Axolotl。** 别自己从零写训练循环。

---

## 📝 练习答案

**用 QLoRA。** 12GB 显存放不下 Llama 3 8B 的 fp16 版本（16GB）。QLoRA 把基础模型压到 4bit（~4GB）+ 适配器 50MB + 训练开销，12GB 够用。
