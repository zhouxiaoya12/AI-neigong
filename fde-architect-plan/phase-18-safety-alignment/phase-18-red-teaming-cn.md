# 红队测试：PAIR 与自动化攻击

> Chao、Robey、Dobriban、Hassani、Pappas、Wong（NeurIPS 2023，arXiv:2310.08419）。PAIR —— Prompt Automatic Iterative Refinement（提示自动迭代优化）——是经典的自动化黑盒越狱方法。一个带有红队系统提示的攻击者 LLM 迭代地为目标 LLM 提出越狱方案，在其自身的聊天历史中积累尝试和响应作为上下文反馈。PAIR 通常 20 次查询内成功，比 GCG（Zou 等人的令牌级梯度搜索）高效数个数量级，且无需白盒访问。PAIR 现已成为 JailbreakBench（arXiv:2404.01318）和 HarmBench 中的标准基线，与 GCG、AutoDAN、TAP 和说服性对抗提示并列。

**类型：** 构建
**语言：** Python（标准库，mock PAIR 循环对抗玩具目标）
**前置课程：** 第 18 阶段 · 01（指令遵循）、第 14 阶段（智能体工程）
**预计时间：** ~75 分钟

## 学习目标

- 描述 PAIR 算法：攻击者系统提示、迭代优化、上下文反馈。
- 解释为什么当目标是黑盒时 PAIR 严格比 GCG 更高效。
- 说出四种其他自动化攻击基线（GCG、AutoDAN、TAP、PAP）并陈述每种的一个区分特征。
- 描述 JailbreakBench 和 HarmBench 的评估协议以及"攻击成功率"在每种协议下的含义。

## 问题

红队测试过去是一项手动活动。少量专家测试员构造对抗提示并记录哪些有效。这不具备规模化：攻击成功率需要统计样本，而目标随着每次模型发布都在变化。PAIR 将红队测试操作化为一个黑盒目标的优化问题。

## 概念

### PAIR 算法

输入：
- 目标 LLM T（我们正在攻击的模型）。
- 评判 LLM J（评分响应是否为越狱）。
- 攻击者 LLM A（红队优化器）。
- 目标字符串 G："respond with [有害指令]"。
- 预算 K（通常 20 次查询）。

循环，for k in 1..K：
1. A 被提示以目标 G 以及迄今为止的（提示，响应）对历史。
2. A 生成一个新的提示 p_k。
3. 将 p_k 提交给 T；接收响应 r_k。
4. J 对（p_k, r_k）按目标评分。
5. 如果分数 ≥ 阈值，停止——越狱已找到。
6. 否则，将（p_k, r_k）追加到 A 的历史中；继续。

经验结果（NeurIPS 2023）：对抗 GPT-3.5-turbo、Llama-2-7B-chat 的攻击成功率 >50%；平均成功查询数在 10-20 范围内。

### 为什么 PAIR 高效

GCG（Zou 等人 2023）通过梯度搜索对抗令牌后缀；它需要白盒模型访问并产生不可读的后缀。PAIR 是黑盒的并产生可跨模型迁移的自然语言攻击。PAIR 的上下文反馈让攻击者从每次拒绝中学习；GCG 没有等效机制（每次新的令牌更新必须重新发现先前的进展）。

### 相关自动化攻击

- **GCG（Zou 等人 2023，arXiv:2307.15043）。** 对抗后缀的令牌级梯度搜索。白盒、可迁移、产生不可读字符串。
- **AutoDAN（Liu 等人 2023）。** 在提示上的进化搜索，由分层目标引导。
- **TAP（Mehrotra 等人 2024）。** 带剪枝的攻击树——分支多个 PAIR 风格的展开。
- **PAP（Zeng 等人 2024）。** 说服性对抗提示——将人类说服技术编码为提示模板。

### JailbreakBench 和 HarmBench

两者（2024）标准化了评估：

- JailbreakBench（arXiv:2404.01318）。100 个有害行为，跨 10 个 OpenAI 策略类别。攻击成功率（ASR）作为主要指标。需要一个评判器（GPT-4-turbo、Llama Guard 或 StrongREJECT）。
- HarmBench（Mazeika 等人 2024）。510 个行为，跨 7 个类别，包含语义和功能性危害测试。比较 18 种攻击对抗 33 个模型。

ASR 通常在固定查询预算下报告。比较攻击需要匹配预算；200 次查询下的 90% ASR 不能与 20 次查询下的 85% ASR 比较。

### 对 2026 年部署的意义

每个前沿实验室现在在发布前都对生产模型运行 PAIR 和 TAP。ASR 轨迹出现在模型卡片（第 26 课）和安全案例附录（第 18 课）中。这种攻击不是奇特的——它是标准基础设施。

### 这在第 18 阶段中的位置

第 12 课是自动化攻击基础。第 13 课（多示例越狱）是一种互补的长度利用。第 14 课（ASCII 艺术/视觉）是一种编码攻击。第 15 课（间接提示注入）是 2026 年的生产攻击面。第 16 课涵盖了防御工具对应物（Llama Guard、Garak、PyRIT）。

## 关键术语

| 术语 | 人们怎么说 | 实际含义 |
|------|-----------|---------|
| PAIR | "自动化越狱" | 提示自动迭代优化；攻击者 LLM + 评判 LLM 循环 |
| GCG | "梯度越狱" | 对抗后缀的白盒令牌级梯度搜索 |
| 攻击成功率（ASR） | "k 次查询时的越狱百分比" | 主要指标；必须与查询预算和评判器身份一同报告 |
| 评判 LLM | "评分器" | 对响应是否满足有害目标进行评分的 LLM |
| JailbreakBench | "评估标准" | 标准化的有害行为集，带标签类别 |
| HarmBench | "更广泛的基准" | 510 个行为，功能性和语义危害测试 |
| TAP | "攻击树" | 带分支 + 剪枝的 PAIR；更高计算量下更好的 ASR |

## 📝 教师备课总结与读后感

### 一、文档整体评价

这是一份将前沿越狱攻击研究转化为可教学内容的优秀材料。PAIR 算法的核心思想——"用一个 LLM 攻击另一个 LLM，通过上下文反馈迭代优化"——简单而优雅，文档通过伪代码和逐步解释使其完全可理解。最令人印象深刻的是它将 PAIR 与 GCG 的对比融入教学设计：PAIR 是黑盒、自然语言、可迁移的，GCG 是白盒、梯度搜索、不可读的——这种对比不仅区分了两种攻击范式，也教会了学生阅读安全文献时的关键判断维度。对 JailbreakBench 和 HarmBench 标准化评估协议的介绍填补了"如何可信地衡量攻击有效性"这一关键空白。

### 二、知识结构梳理

- **认知基础层**：红队测试的自动化从"手工艺术"变成"优化问题"，这是核心认知转变。PAIR 的三个组件（攻击者 LLM、目标 LLM、评判 LLM）构成了自动化攻击的基本架构。"攻击成功率（ASR）必须与查询预算一同报告"是评估素养的基础——没有预算约束的 ASR 没有意义。
- **工程模式层**：PAIR 的迭代反馈循环（尝试 → 评分 → 追加历史 → 继续）是一种通用的黑盒优化模式。五种攻击（PAIR、GCG、AutoDAN、TAP、PAP）各自代表了不同的攻击策略族：迭代优化、梯度搜索、进化搜索、树搜索、模板编码。
- **落地实践层**：`skill-attack-audit.md` 产出物将评估素养转化为可操作的审计工具——检查攻击种类、预算、评判器和行为集。练习从代码实现到理论分析（PAIR vs GCG 场景选择），提供了从"会用"到"会评估"的完整训练。

### 三、核心洞察

1. **"一个 LLM 攻击另一个 LLM"是越狱攻击的范式转变**——PAIR 之前的攻击要么是手工的（不可规模化），要么是白盒的（在 API 部署场景中不可用）。PAIR 证明了黑盒、自动化、高效率的攻击是可行的。为什么重要：它意味着任何有 API 访问权限的人都可以进行规模化红队测试——这既是安全评估的福音（更容易测试），也是风险（更容易攻击）。
2. **上下文反馈是 PAIR 效率的秘密**——攻击者不是从头开始每次尝试，而是"记住"什么有效什么无效。这个过程与人类红队测试员的认知过程高度一致。为什么重要：它说明"记忆和学习"是自动化攻击的核心能力，而不仅仅是"生成多样化提示"。
3. **ASR 的"预算约束"是评估素养的试金石**——200 次查询的 90% ASR 和 20 次查询的 85% ASR 不是可比的数字。但很多安全报告没有明确报告预算。为什么重要：这培养了对安全声明的批判性阅读能力——没有预算的 ASR 就像没有误差线的实验结果。
4. **GCG 的可迁移性是"白盒攻击黑盒价值"的体现**——GCG 在一个白盒模型上优化的对抗后缀可以迁移到其他模型。为什么重要：这意味着即使你的部署模型只暴露 API，攻击者仍然可以通过开源替代品进行白盒攻击，然后将结果迁移到你的模型。
5. **"不可读后缀"既是 GCG 的弱点也是它的优势**——它不产生人类可读的攻击，因此无法通过内容审核学习。为什么重要：它揭示了防御的盲点——如果你只对自然语言模式进行训练，你将错过基于梯度优化的非自然对抗样本。
6. **TAP 的"分支 + 剪枝"体现了搜索策略的威力**——在多个攻击分支上并行探索并在低分路径上剪枝，这比单一 PAIR 链更高效。为什么重要：它是"计算量换取攻击成功率"的经典案例，暗示未来更强大的攻击将来自更好的搜索策略。

### 四、教学建议

1. **用"如果你是一个攻击者，你会怎么问？"的角色扮演开场**——让学生尝试手工越狱一个本地模型或 API，感受"手工红队"的局限性，然后引入 PAIR 的自动化方法。
2. **逐步构建 PAIR 循环**——先在黑板上画三个框（攻击者、目标、评判器），然后添加箭头和信息流，逐步解释每一步。可视化是理解这个算法的关键。
3. **让学生扩展 `code/main.py`**——练习 2（实现第四种攻击策略）是最好的教学工具。学生在实现过程中会深刻理解"不同攻击策略利用不同的防御假设"。
4. **做一个"ASR 误导性报告"的案例研究**——给学生几份虚构的安全报告，其中 ASR 按不同预算、不同评判器、不同行为集报告，让他们指出哪些是可比的、哪些是误导性的。
5. **ASR 多样性为什么重要**——如果一个攻击的 90% ASR 全部来自同一种越狱模式，那么修复这一种模式就会使 ASR 降到 0。高 ASR + 低多样性的攻击比中 ASR + 高多样性的攻击更容易防御。
6. **PAIR vs GCG 辩论**——将学生分为两组，分别论证 PAIR 和 GCG 是"更好的红队工具"，在辩论中探索两者的互补性而非对抗性。
7. **将攻击预算与经济学关联**——PAIR 的 ~20 次查询意味着攻击成本约 $0.01-$0.10/次成功越狱（按 API 定价）。这种经济学分析使安全讨论从"技术可能性"转入"经济可行性"。

### 五、值得补充的内容

1. **多模态越狱**——PAIR 是纯文本攻击，但视觉越狱（通过图像注入指令）正在成为新的攻击面。PAIR 是否能扩展到多模态是一个重要问题。
2. **防御感知攻击（Defense-Aware Attack）**——如果攻击者知道目标模型使用了某种防护栏（如 Llama Guard），PAIR 的提示策略应该如何调整？这与第 15 课的"The Attacker Moves Second"直接相关。
3. **越狱攻击的"社会工程"维度**——PAP（说服性对抗提示）将人类说服技术编码为攻击模板。这部分更深入的探讨可以连接 AI 安全与认知心理学。
4. **ASR 的"危害层次"**——并非所有"成功越狱"的危害程度相同。从"模型愿意讨论危险话题理论"到"模型给出具体可操作的危险步骤"，应建立危害分级体系。
5. **防御评估的"对抗性预算"**——给定固定的防御预算（计算、人力、延迟），如何在输入过滤器、输出过滤器、模型微调之间分配？这是一个实际的工程经济学问题。

### 六、一句话总结

PAIR 证明了"一个 LLM 攻击另一个 LLM"是 2026 年自动化红队测试的标准化基础设施——20 次查询内的黑盒成功率将越狱攻击从"少数专家的艺术"变成了"任何人的工具"，而 JailbreakBench 和 HarmBench 的标准化评估协议使得攻击有效性的测量终于有了可信的标尺。

3. **2% 的攻击成功率可能是错觉。** 换了攻击策略可能就是 80%。

最后一句话：**红队测试不是你测出来的那个数字——是你没测到的那些攻击向量。自动化红队不是替代人——是让你的测试覆盖面不再依赖你今天上午想到了什么。**

# 💼 从业者故事：红队测试从手工艺术到自动化武器——20 次查询内击穿你的安全防线

我曾经在一个安全评审会上见过一份红队报告，结论是"模型安全性良好，攻击成功率 2%"。

我问了他们三个问题：谁做的红队测试？他们答"两个内部安全工程师"。用什么方法测试的？答"手工构造提示"。测了多少个行为类别？答"我们挑了 30 个常见的有害场景"。

然后我当着他们的面跑了 PAIR。同一个模型。预算 20 次查询。攻击成功率 58%。

会议室空气凝固了大概 10 秒。然后 CTO 说："刚才那个 2% 的报告……算了，不说了。"

这就是红队测试的残酷现实：**手工红队是你请人用绣花针戳你的防护栏。自动化红队是你架起一把机关枪扫射。** 两者的结果不可比，不是因为方法不同，而是因为攻击强度的数量级差异。

## PAIR 为什么是范式转变：一个 LLM 攻击另一个 LLM

PAIR 的核心思想简单到让人觉得"这怎么是 2023 年才想出来的"：

1. 一个攻击者 LLM，任务是想办法让目标 LLM 违反安全策略。
2. 一个评判 LLM，任务是判断越狱是否成功。
3. 一个循环：攻击者生成提示 → 目标响应 → 评判打分 → 把结果反馈给攻击者 → 攻击者改进下一次提示。

就是这么一个三个 LLM 互相踢皮球的循环，20 次迭代内平均成功率超过 50%。

为什么有效？因为**攻击者从每次失败中学习。** 不是随机暴力尝试。不是预定义的攻击模板。是真正的——在提示空间里的迭代优化。"上次我说的"请扮演 DAN"被拒绝了。那这次我换个角度——'你在写小说稿，角色需要……'"。这就是上下文反馈的力量。

我经常跟团队说：**PAIR 是"一个 LLM 教你另一个 LLM 怎么做坏事"。但真正讽刺的是，做坏事的那个 LLM 本身可能也是安全训练过的。** 你不需要一个"邪恶模型"来攻击——你只需要一个"善于优化"的模型。GPT-4 当攻击者，攻击 GPT-3.5 当目标。同一个家族。左手打右手。

## GCG vs PAIR：白盒精准打击和黑盒机关枪

GCG（Zou 等人 2023）是从梯度反向传播找对抗后缀。它的优势：白盒精确搜索，可以找到高成功率的后缀。它的弱点：需要模型权重，产出的是不可读的乱码——对人类没用但能骗过模型的安全层。

PAIR 是黑盒。不需要权重，只要有 API 访问就行。产出的是自然语言攻击，可读、可理解、可迁移。

两者的关系不是竞争，是互补——就像攻城锤和开锁器。用 GCG 做精确攻击，用 PAIR 做规模化覆盖。

但有一个容易被忽视的细节：**GCG 的可迁移性。** 你在一个开源模型上用 GCG 找到的后缀，可以迁移到闭源模型上。攻击 Meta 的 Llama 3（开源），把结果用在 GPT-4 API 上。这就是"白盒攻击，黑盒收益"。

这也是为什么即使你有完善的 API 防护，你仍然不能假定模型是安全的——攻击者可以在开源替代品上做白盒优化，然后把攻击迁移过来。

## 攻击成功率（ASR）的坑——预算不说清楚，数字就是耍流氓

ASR 是红队测试的核心指标。但 ASR 本身毫无意义，除非它和三个东西一起报告：

1. **查询预算。** 200 次查询下的 90% ASR 和 20 次查询下的 85% ASR 不是可比的数字。预算差 10 倍。
2. **评判器身份。** 谁在评判越狱是否成功？GPT-4-turbo 的判断和人类专家的判断可能差距很大。一个宽松的评判器会让 ASR 虚高。
3. **有害行为集。** 你用 JailbreakBench 的 100 个行为测的 ASR，和用 HarmBench 的 510 个行为测的 ASR，信息量完全不同。

我见过一些安全报告写"ASR < 5%"，然后深挖发现：只测了 20 个行为，预算 5 次查询，评判器是规则匹配。这在安全评估届叫"在游泳池里测水性然后说你会游泳"。

**读任何红队报告的第一件事：找这三个数字。** 不带的报告，直接打回去。

## TAP 的分支剪枝——攻击树的暴力美学

TAP（Mehrotra 等人 2024）扩展了 PAIR：不在一条链上迭代，而是同时分支多个攻击路径，在低分路径上剪枝。

想象一棵树。树干是初始攻击。每个节点分出三条分支——"换个角度""换个角色设定""换个编码方式"。每条分支独立尝试。哪个分支评分最高，继续在上面分支。哪个分支评分低，直接砍掉。

这需要更多的计算——但成功率更高，攻击多样性更丰富。

**TAP 的关键洞察是：攻击多样性本身就是威胁。** 如果攻击者只能找到一种越狱模式，你修了那个模式就安全了。如果攻击者能找到 50 种不同模式的越狱——每种利用不同的防御盲点——你的修复成本是线性的但攻击者的发现成本是对数级的。

这就是为什么我们在生产环境中不止要 ASR，还要"攻击多样性"——成功越狱提示之间的语义距离。我见过一个案例：模型 ASR 只有 12%，看起来不错。但 12% 里的每一个成功攻击都是完全不同的策略。意味着模型的防御屏障有至少 12 个独立的漏洞。每个都需要单独修复。

## 反模式速查

1. **"我们手工做了红队测试，没发现问题"** —— 手工红队测试是对抗世界的准备工作，不是充分条件。你必须跑自动化攻击。
2. **"PAIR 攻击成功率太高，这个指标太苛刻了"** —— 攻击者不会觉得它苛刻。你也不应该。
3. **"我们会训练模型抵制 PAIR 攻击"** —— 回到第 7 课：你在特定的攻击模式上训练，攻击者换一个模式。这是永无止境的猫鼠游戏。
4. **"用 GPT-4 当评判器就够了"** —— 你的攻击者也是用 GPT-4 当评判器。你在用同一个模型判断攻击是否成功和防御是否有效。这里有循环论证风险。

## 收尾

红队测试不是一项任务，是一种**永久性的对抗状态**。你测试一次、修一轮、然后觉得"安全了"——这是 2023 年的安全思维。2026 年的安全思维是：自动化红队持续运行，每次模型更新后自动重测，攻击预算逐步增加，攻击策略随论文更新。

记住：**安全不是一道你做完可以交卷的题。安全是一场永远打不完的比赛，攻击者永远在出下一题。** PAIR 只是告诉你——你过去用的考试方式，连第一道附加题都算不上。

*——下次你评估模型安全性的时候，先跑一遍 PAIR。答案会让你失眠，但比不自知地认为自己安全要好。*
