# DPO：跳过 RLHF 的复杂，直接优化人类偏好

> **原文：** DPO: Direct Preference Optimization  
> **预计时间：** ~25 分钟  

---

## 🚗 开场翻车：三个模型、三个阶段、调参调到崩溃

RLHF 好用。但三个模型（SFT + 奖励 + 策略）、PPO 不稳定、KL 系数要调——小超参数变化导致训练发散。

2023 年 5 月，Stanford 团队发了 DPO 论文。核心洞察：**你不需要单独的奖励模型。语言模型本身的 token 概率在数学上等价于奖励信号。**

---

## 🧠 DPO 怎么跳过 RLHF 的

```
RLHF: SFT → 训练奖励模型 → PPO 优化策略（3个模型，复杂）
DPO:  SFT → 直接在偏好对上优化（1个模型，简单）
```

**DPO 直接让模型从"人类选 A 不选 B"的偏好对中学习。** 不需要强化学习，不需要奖励模型，不需要 KL 调参。

---

## 📊 RLHF vs DPO

| | RLHF | DPO |
|---|---|---|
| 模型数量 | 3 个 | 1 个 |
| 训练循环 | PPO + 奖励模型 | 单个监督学习 |
| 稳定性 | 低（PPO 敏感） | 高 |
| 实际使用 | InstructGPT/ChatGPT | Llama 3、Zephyr-7B |

Zephyr-7B 是首批大规模用 DPO 的模型，在多个基准上匹配或超越完整 RLHF 训练的模型。Meta 在 Llama 3 对齐流水线中用了 DPO。

---

## 🏆 焊死在脑子里的东西

1. **DPO = 跳过奖励模型，直接在偏好对上训练。** 一个损失函数替代整个 RLHF 流水线。
2. **数学基础：最优奖励函数 = 模型自己 token 概率的比值。**
3. **RLHF 没有被淘汰——但在很多场景 DPO 更简单、更稳定。**

---

> **RLHF 用了三个模型和一个不稳定的 PPO。DPO 用一个模型和一个损失函数做到了同样的事。有时候最优雅的解决方案就是删代码。**
