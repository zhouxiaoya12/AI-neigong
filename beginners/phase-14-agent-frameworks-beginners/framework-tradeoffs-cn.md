# Agent 框架选型：LangGraph vs CrewAI vs AutoGen vs Agno

> **原文：** Framework Tradeoffs  
> **预计时间：** ~20 分钟  

---

## 🚗 开场翻车：三天后发现框架在跟你打架

你选了个框架。三天后——CrewAI 的角色让你写研究员输出，但"作者"看不懂研究者给的格式。AutoGen 的对话很好，但崩了没法恢复。LangGraph 让你画图，但你还没动手就知道 Agent 会干嘛。

**不是"哪个框架最好"。是"哪个框架的抽象匹配你的问题形态"。**

---

## 📊 2026 年四大框架

| 框架 | 核心抽象 | 状态处理 | 最适合 | 最不适合 |
|------|---------|---------|--------|---------|
| **LangGraph** | 状态图（节点+边+类型化状态） | 每步检查点 | 带显式状态的流水线 | 松散的头脑风暴 |
| **CrewAI** | 角色团队（goal+backstory） | 字符串流 | 角色扮演型工作流 | 复杂分支 |
| **AutoGen** | Agent 对话 | 对话历史 | 师生/提案-评审对话 | 确定性 DAG |
| **Agno** | 单体 Agent + 工具 | 内置存储驱动 | 快速单 Agent + 轻量团队 | 深度分支图 |

---

## 🎯 一句话选型指南

```
任务形态像流水线（步骤1→2→3，状态要持久） → LangGraph
任务形态像团队协作（角色清楚，分工明确）    → CrewAI
任务形态像对话（互相质疑，达成共识）        → AutoGen
任务形态像单兵作战（一个Agent+工具就够了）   → Agno
```

---

## ⚠️ 状态是你选框架最终翻车的地方

- **LangGraph：** 类型化状态 + 每步检查点 → 崩溃恢复精确。
- **CrewAI：** 状态是字符串流→ 跨重启你得自己存。
- **AutoGen：** 状态是对话历史→ 跨重启能回放对话，但工作流状态自己管。
- **Agno：** 存储驱动内置→ 会话自动持久化，但不是图级检查点。

---

## 🏆 焊死在脑子里的东西

1. **框架的核心抽象 = 你在白板上画的那个东西。** LangGraph 画图，CrewAI 画组织架构，AutoGen 画聊天。
2. **别选"最好"的框架——选抽象匹配你问题的框架。**
3. **状态持久化是生产选型的第一考量。** 崩了能精确恢复 > 看着漂亮。

---

> **四种框架，四种心智模型。不是谁好谁坏——是你的问题长成什么样。流水线用 LangGraph，角色团队用 CrewAI，对话用 AutoGen，轻量用 Agno。**
