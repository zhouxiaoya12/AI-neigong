# 高级 RAG：混合搜索、重排序、分块策略

> **原文：** Advanced RAG  
> **预计时间：** ~25 分钟  

---

## 🚗 基础 RAG 的三次翻车

**翻车 1：** "上季度收入多少？"语义搜索返回了"收入策略"、"收入预测"——全是"收入"相关，但没一个含数字。正确答案用的是"earnings"而不是"revenue"。

**翻车 2：** "哪个团队满意度提升最大？"答案分散在 5 个团队的报告中——没有单个文档块包含完整答案。

**翻车 3：** 200 万个文档块。正确答案在 #1,847,293。前 5 名返回了 #14, #89K, #1.2M, #44, #901K。在向量空间里"接近"，但没一个有用。

---

## 🔧 四种修复

### 1. 混合搜索：语义 + 关键词

```
语义搜索：理解"取消订阅" ≈ "终止计划"（词不同但意思同）
关键词搜索：精确匹配 "Error code E-4021"（语义搜索可能漏掉）
混合搜索 = 两者都跑 + 合并结果
```

### 2. 重排序（Re-ranking）

```
第一步：快速检索 top 100 候选块（向量搜索，快但粗）
第二步：用更强的 Cross-Encoder 对 100 个候选重新打分
第三步：返回重新排序后的 top 5
```

**Cross-Encoder 把查询和文档一起编码**，比 Bi-Encoder 精确但慢——只在 top-K 候选上用。

### 3. 查询转换

| 技术 | 做法 |
|------|------|
| **HyDE** | 先生成一个"假设答案"，再拿答案去搜 |
| **多查询** | 把一个问题改写成多个角度去搜 |
| **回退** | 搜不到就扩大范围/降低阈值 |

### 4. 更好的分块

- **语义分块** — 按段落/章节边界切，不按固定长度
- **父子分块** — 大块给上下文，小块给检索精度

---

## 🏆 焊死在脑子里的东西

1. **向量相似 ≠ 回答问题有用。** 基础 RAG 的天花板。
2. **混合搜索 = BM25 + 向量搜索。** 精确匹配和语义理解互补。
3. **Cross-Encoder 重排序 > 提高 top-K。** 在少量候选上做精确打分。

---

> **基础 RAG 跑通只需要一天。高级 RAG 让它从"有时对"变成"几乎每次都对"。混合搜索、重排序、查询改写——三个技术，性价比最高的升级路径。**
