# LangGraph：Agent 作为有状态图

> **原文：** LangGraph: Stateful Graphs & Durable Execution  
> **预计时间：** ~20 分钟  

---

## 🧠 核心抽象：Agent = 状态机

Agent 不是"一个循环"——是一张图：节点（步骤）+ 边（转换）+ 状态（每一步的数据）。

```
START → classify → refund / bug / sales → done → END
                      ↑--- 条件边 ------↓
```

---

## 🔧 四大能力

### 1. 持久化执行 ⭐

**每一步之后自动保存状态到检查点存储。** 第 38 步崩溃 → 从第 37 步恢复，不从头开始。

### 2. 流式传输

每个节点产出增量 → UI 实时更新。

### 3. 人类在环

关键节点暂停 → 人类审查/修改状态 → 继续。检查点存储让这变得简单。

### 4. 记忆

短记忆（当前线程）+ 长记忆（跨线程）→ 两级记忆管理。

## 📊 LangGraph vs 其他框架

| | LangGraph | CrewAI | AutoGen |
|---|---|---|---|
| 抽象 | 状态图 | 角色团队 | 对话 |
| 状态 | 类型化 + 检查点 | 字符串流 | 对话历史 |
| 恢复 | 精确到步 | 重跑任务 | 重跑对话 |
| 适合 | 需要精确控制的流水线 | 角色扮演型团队 | 多 Agent 对话 |

---

## 🏆 焊死在脑子里的东西

1. **LangGraph = 把 Agent 当状态机。** 节点做工作，边做路由，状态不可变。
2. **检查点 = 每步后保存。** 崩溃恢复精准到步。
3. **生产用户：Klarna、Uber、J.P. Morgan。** 不是玩具框架。

---

> **LangGraph 不是"又一个 Agent 框架"——是"Agent 需要持久状态"这一命题的工程答案。选了它意味着你不再为"崩了怎么恢复"写胶水代码。**
