# 缩放定律（Scaling Laws）

> 2020年的 Kaplan 论文说：模型越大，损失越低。2022年的 Hoffmann 论文说：你一直训练不足。计算开销分到两个桶里——参数和令牌——而这个分配并不显然。

**类型：** 学习
**语言：** Python
**前置要求：** Phase 7 · 05（完整 Transformer）、Phase 7 · 07（GPT）
**时间：** 约45分钟

## 问题所在

当你有 C FLOPs 的训练计算量并想要最佳模型时，面对两个旋钮：

1. **多少参数（N）？** 更大的模型，更高容量。
2. **多少训练令牌（D）？** 更多数据，更好利用容量。

FLOPs 近似为 `6 × N × D`。你可以推高 N 压低 D，或者推高 D 压低 N。哪种更好？

2022年以前，答案是"使劲推 N"。GPT-3（2020）是175B参数训练在约300B令牌上。比例约 1.7 令牌/参数。Kaplan 缩放定律支撑了这一做法。

Hoffmann 等人（2022）训练了一组名为 Chinchilla 的小型模型家族，发现了不同结果：最优比例更接近 **20 令牌/参数**。GPT-3 训练不足约10倍。Chinchilla（70B参数、1.4T令牌）在每个基准上都超越了 GPT-3（175B、300B令牌），且推理成本仅为其 40%。

2026年是 Chinchilla 的世界——但有一个重要转折。Llama 3 8B 训练在 15 万亿令牌上，比例是 1875 令牌/参数——是 Chinchilla 最优的94倍。对于将被大规模使用的模型，推理成本比训练成本更重要，因此超越 Chinchilla 的过度训练来换取更小的可部署体积是2026年的默认策略。

## 核心概念

![Chinchilla 曲线：不同 N/D 比例下损失与计算量的关系](../assets/scaling-laws.svg)

### Hoffmann 定律

来自 Chinchilla 论文，损失遵循：

```
L(N, D) = A / N^α + B / D^β + E
```

- `N` = 参数（非嵌入参数）。
- `D` = 训练令牌。
- `α ≈ 0.34`、`β ≈ 0.28`（大致对称）。
- `E ≈ 1.69`，不可约损失下限。
- `A ≈ 406`、`B ≈ 411`。

两项随着你的扩展相互权衡。在固定计算 C = 6ND 下对 N 求导并求解：

```
N_opt ≈ 0.6 × (C/6)^0.5
D_opt ≈ 0.6 × (C/6)^0.5
D_opt / N_opt ≈ 20
```

计算最优：20 令牌/参数。

### 为什么还要过度训练

Chinchilla 最优最小化的是训练损失/训练 FLOP。但你只付一次训练成本；推理成本是永远的。

对于每月服务一万亿令牌的聊天机器人，推理成本主导总成本。Llama 的做法：训练更小、更久。8B 参数在 15T 令牌上是深度推理优化的：

- 适合消费级 GPU。
- 延迟是 Chinchilla 最优 70B 的几分之一。
- 质量在大多数任务上足够接近。

DeepMind 的 2024 论文（"过度训练是新最优"）对此做了形式化。对于推理主导的工作负载，正确比例取决于服务量，大约在 100-500 令牌/参数之间。

### 涌现 vs 平滑

主张：某些能力（算术、多步推理、思维链跟随）在某个规模上突然"涌现"。

Schaeffer 等人（2023）论证这是一个测量伪影：涌现指标使用不连续的评分（精确匹配、阈值准确率），隐藏了底层 logit 中的平滑改进。连续指标（交叉熵）显示出平滑曲线。

2026年的共识：通过连续损失的预测是可靠的。基准跳升通常是评分器的伪影。在连续指标上规划预算。

### 2026年的图景

缩放定律仍然有效，但：

| 因素 | 变化方式 |
|------|---------|
| 数据质量 | 筛选"好"令牌（Phi 风格）以 >2× 的有效计算量移动曲线 |
| MoE | 总参数与活跃 FLOPs 解耦；基于活跃 FLOPs 的缩放定律 |
| 后训练 | 某些能力（指令遵循、代码）通过 SFT+RLHF 比预训练移动更多 |
| 多模态 | 图像 + 文本令牌一起缩放；每个模态有单独的曲线 |
| 合成数据 | 模型生成训练数据；有效计算量可以复合增长 |

Muon 优化器（Kimi Moonlight, 2024）在匹配数据上展示了相比 AdamW 约 2× 的有效计算增益。一些2026年的训练运行为默认使用 Muon。这改变了缩放定律中的绝对常数，而不是其形状。

## 动手构建

参见 `code/main.py`。我们实现 Chinchilla 损失方程，并对多个计算预算求解计算最优的 `(N, D)`。

### 步骤 1：Chinchilla 损失

```python
def chinchilla_loss(N, D, A=406.4, B=410.7, alpha=0.34, beta=0.28, E=1.69):
    return A / N ** alpha + B / D ** beta + E
```

在固定 `C = 6ND` 下绘制 `L` 在 `(N, D)` 上的等高线。找到最小值。

### 步骤 2：计算最优前沿

对于从 `1e17` 到 `1e25` FLOPs 的计算预算，找到使损失最小化的 `(N, D)`，约束为 `6ND = C`。验证比例 `D/N ≈ 20`。

### 步骤 3：过度训练成本

计算训练一个10倍小型模型（最优 N 的 1/10，最优 D 的10倍）所付出的额外损失。报告与之交换的推理 FLOPs 节省（与 N 成正比）。

### 步骤 4：对比真实模型

代入 GPT-3、Chinchilla、Llama 3 8B、DeepSeek-V3（活跃参数）的已知 `(N, D)` 对，比较预测损失与报告损失。

## 实战使用

你不太可能自己训练一个前沿模型。但缩放定律告诉你：

1. **你的微调是否有足够的数据。** 如果你的任务特定数据低于基础模型的 20 令牌/参数，预期在某些损失地板附近饱和。
2. **是否选择更大的基础模型。** 如果你把所有预算花在推理上，优先选择更小、训练更久的模型。
3. **收益在哪里递减。** 超过 Chinchilla 最优的1000倍后，对数损失的变化变成噪声。

**2026年的研究方向：**

- **数据受限机制。** 网络只有有限的高质量令牌（过滤后约5~10万亿英语）。前沿预训练正在接近这个上限。合成数据、多语言、多模态和 RLHF 规模化的微调是下一波杠杆。
- **计算乘数技巧。** Muon 优化器、MoE、更好的数据筛选——每个都改变绝对常数，而非渐近线。
- **RL 的缩放定律。** 开放问题。早期证据表明 RL 样本中存在幂律，但指数与预训练非常不同。

## 关键术语

| 术语 | 人们怎么说 | 实际含义 |
|------|-----------|---------|
| 参数（N） | "模型大小" | 非嵌入权重计数；决定容量。 |
| 令牌（D） | "训练数据" | 见过的训练令牌数；决定参数被利用得多好。 |
| 计算量（C） | "花费的 FLOPs" | 标准 Transformer 近似 `6 × N × D`。 |
| Chinchilla 最优 | "D/N ≈ 20" | 最小化预训练每 FLOP 损失的比例。 |
| 过度训练 | "超越 Chinchilla" | 花费额外训练 FLOP 节省推理 FLOP；D/N >> 20。 |
| 不可约损失 | "地板" | 缩放定律中的 `E` 项；数据本身的熵。 |
| 涌现能力 | "规模处的突然跳跃" | 通常是评分器伪影；连续损失是平滑的。 |
| 有效计算量 | "训练效率乘数" | 更好的数据/优化器/架构使每个 FLOP 走得更远。 |

## 📝 教师备课总结与读后感

### 一、文档整体评价

这可能是本阶段最重要的一课——不是因为技术难度，而是因为它从根本上改变了一个工程师的预算思维。从 Kaplan 到 Chinchilla 到 Llama 的"过度训练"，这篇教材用一条清晰的曲线讲述了整个业界如何重新理解"参数"和"数据"的关系。特别值得称赞的是它没有停留在"D/N=20 是最优的"就结束，而是抛出了"那为什么 Llama 3 用了 1875 倍？"这个直指工程本质的问题。

### 二、知识结构梳理

- **基础层**：幂律公式 L(N,D) = A/N^α + B/D^β + E、求导得到最优 N/D 比的数学过程、FLOPs 近似 6ND
- **模式层**：Kaplan（D/N≈1.7，数据少参数大）vs Chinchilla（D/N≈20，平衡分配）vs 过度训练（D/N≈100-2000，推理成本优先）
- **应用层**：训练预算决策（选多大的模型、准备多少数据）、推理成本主导的宏观策略（小模型长训练）、涌现 vs 平滑的测量方法论之争

### 三、核心洞察

1. **缩放定律不是"精确公式"，是"分配原则"**：A、B、α、β 的具体值会变（受数据质量、优化器、架构影响），但 L(N,D) 的两项竞争结构是不变的——你总要在"容量"和"利用率"之间分配计算量。
2. **Chinchilla 最优回答的是"训练预算怎么花"**——如果你只关心训练损失。但大多数生产场景的训练预算只是总预算的 1-5%，推理预算才是主体。这时候 Chinchilla 是错的。
3. **Llama 3 8B 的 D/N=1875 不是"背叛"Chinchilla，是"超越"**：这个数字本身说明——如果把推理成本计入目标函数，最优点会偏向"小而久"的方向。
4. **涌现的真相是"连续函数通过不连续度量看起来像跃迁"**：Schaeffer 2023 可能是近年最具破坏力的机器学习论文之一——它让几十篇"涌现能力"论文的结论变得可疑。
5. **"有效计算量"的概念比缩放定律本身更重要**：数据筛选、MoE、Muon——这些技巧乘以训练效率的常数因子。在一个幂律世界里，常数因子的效果是指数级放大的。
6. **不可约损失 E 是墙上印着的最终标记**：模型再大、数据再多，只能逼近数据的熵。当你的训练损失接近 E，再多的计算量都没用——这就是自然语言的下界。
7. **合成数据是打破"数据墙"的唯一出路**：高质量人类文本总量有限（~5-10T 英语 token），而 Chinchilla 最优对一个 1T 参数的模型需要 ~20T token。合成数据或成为新一代缩放定律的核心变量。

### 四、教学建议

1. **不要从公式开始，从"预算分配困境"开始**：给一个场景——你有 1e23 FLOPs 的预算，要怎么分给 N 和 D？先让学生凭直觉猜，再展示 Chinchilla 的答案。
2. **手动画 L(N,D) 的三维曲面**：X 轴=log N, Y 轴=log D, Z 轴=Loss。找出 C=6ND 的约束对角线上的最低点——这个视觉比公式更有说服力。
3. **做"我（学生）自己的 Chincilla"实验**：用五个不同配置的小模型（改 N 和 D），测量训练损失，拟合 α 和 β。这比读论文更能建立直觉。
4. **对比 Kaplan 和 Hoffmann 的论文方法差异**：为什么 Kaplan 会得出 D/N≈1.7？两者在"参数"定义、损失定义上的细微差别如何导致了方向性差异。
5. **用生产数据讲故事**：准备一个表列出 GPT-3、Chinchilla、Llama 3 8B、DeepSeek-V3 的 (N, D, N/D, 推理成本/令牌) —— 趋势一目了然。
6. **把"涌现"辩论作为一个科学研究方法案例**：Schaeffer 2023 不仅是一个技术结论，更教给学生"当你在一个不连续指标上看到 '突然'，先检查你的指标是不是连续的"。
7. **留一个 "怎么选学习率 schedule"的悬念**：本课讲 CHinchilla 告诉你 N/D 比例，但还有更重要的问题——给定这个 N/D，在无限时长稳定训练和有限步长的 cosine 衰减之间怎么选——这是 Warmup-Stable-Decay 和 WSD LRS 的最新争议，可以留到后续讨论。

### 五、值得补充的内容

1. **Scaling Law 对 Fine-tuning 的指导**：如果你在微调一个模型，你的微调数据量应该怎么算？应该是 20× 微调参数？还是 20× 整个模型参数？目前没有共识，但可以讨论不同场景的经验。
2. **DeepSeek-V3 的 WSD 学习率策略**：DeepSeek 的论文提出了 Warmup-Stable-Decay 学习率安排，配合 MoE+MLA 的超高 D/N 训练——这正在成为新的范式。
3. **"数据墙"的定量分析**：CommonCrawl 过滤后的高质量 token 到底有多少？Epoch AI 的"数据墙"估算和 Anthropic 的 scaling-monosemanticity 交叉阅读。
4. **MoE 的缩放定律改写**：活跃参数 vs 总参数的解耦怎样改变 C = 6ND 的近似？活跃 FLOPs 的概念对 MoE 更适用。
5. **多模态缩放定律**：文本+图像 token 的混合缩放——视觉 token 和文本 token 的"有效信息量"不同，如何折算？

### 六、一句话总结

缩放定律不是告诉你"模型应该多大"，而是告诉你"你的预算在参数和数据之间的分配有没有均衡"——Chinchilla 找到了训练预算的均衡点，Llama 找到了推理成本的均衡点，下一代需要找到的均衡点是"合成数据自举+推理成本最小化"。

# 🎓 Agent 架构课：缩放定律——为什么你训练模型的方式可能是错的？

**副标题：从"越大越好"到"分配才是艺术"的预算思维革命**

---

先问一个问题——

你有 1e24 FLOPs 的计算预算。你可以训练一个 300B 参数的模型，只给它 0.6T 令牌的数据。也可以训练一个 50B 参数的模型，给它 3.3T 令牌的数据。你选哪个？

……

如果你觉得"300B 的模型肯定更强"，那我告诉你——**按 Chinchilla 缩放定律，50B + 3.3T 这个组合的损失更低。** 你是错的。而且2022年以前，几乎整个业界都是错的。

GPT-3 就是"前者"的代表：175B 参数，300B 令牌。比例 1.7 令牌/参数。它开创了一个时代。但它的损失并非最优——Hoffmann 2022 证明了，在同样的训练 FLOPs 下，一个 70B 参数、训练在 1.4T 令牌上的模型能以更低损失胜出。这就是 Chinchilla。

让我给你两条路：

**路径一：Kaplan 的世界。** "参数就是一切。" 使劲堆参数，数据差不多够用就行。GPT-3、Gopher——这些模型的 N/D 比在 1-2 之间。它们的推理成本高得离谱。一个 175B 的训练预算，你得为每个推理令牌付全价。

**路径二：Chinchilla 的世界。** "参数和数据要平衡。" D/N ≈ 20。你把同样的训练计算量分配得更均匀。结果：推理成本仅为路径一的 40%，但质量更高。这是个不可思议的结论——花得更少，得到更多。

我在 2023 年之后就没有在任何项目里用过路径一。不是因为 Kaplan 错了——"错了"是太强的词——而是因为当你有 Chinchilla 这个"同样的 FLOPs 获得更好结果"的方案时，Kaplan 的方案就不再是"一个可行的选项"，它是"一个次优的选项"。

但我必须马上告诉你——路径二在 2026 年也不是默认。默认是一个叫"过度训练"的新路径。

**路径三：过度训练的世界（Llama 3）。** "推理成本才是主体。" D/N >> 20——往往是 200-2000。Llama 3 8B 训练在 15T 令牌上，比例 1875:1。这是在说：**我宁愿在训练上多花 94 倍 Chinchilla 应该花的 FLOPs，只要推理时我的模型够小、够快、够便宜。** 因为训练只花一次钱，推理永远在花钱。

现在，这个逻辑能不能无限推？不能。**你训练一个 500M 参数模型，给它无限数据——它最后会逼近一个不可逾越的损失下限 E。** E 大约是 1.69 的交叉熵（对于英文 web 文本）。这是由数据本身随机性决定的"墙"。再多参数也不能穿透墙——墙那边是完美预测 token 的幻觉。

这才是缩放定律真正教给我的东西，不是数字、不是公式，而是一个思维框架：**你总有两个 knobs——容量（N）和利用率（D）。你的预算（C）在这两个 knobs 之间的分配比例是所有其他决定的基座。** 在这个基座之上，数据质量、优化器、架构——每一个最多给你一个 2-5 倍的常数因子增益。常数因子在幂律里被指数放大，所以它们极其重要。但基座——分配比例——决定了你在哪个数量级上。

**一个具体的反模式我必须点名批评。** 我遇到过不止一个团队在微调时直接照搬 Chinchilla 的 D/N=20 规则。微调不是这样的——微调的"有效数据规模"不是你的微调数据集，而是预训练数据。你对已经吸饱了 15T 令牌的 Llama 3 做 1M 令牌的微调——那 1M 令牌的数据量 N/D 完全不是 Chinchilla 的概念。不要用预训练的缩放定律直译微调场景。**微调的门槛不是"数据够不够"，而是"微调的分布和预训练的分布漂移多大"——漂移越大，需要的数据越多。**

还有一个反模式：**把"涌现能力"当做一个科学事实来规划预算。** 如果你在做一个产品 roadmap，你说"我们需要把模型从 7B 扩展到 70B，因为 70B 时某个能力会'涌现'"——我会直接拒绝这个预算。Schaeffer 2023 在统计学上证明了：大多数所谓的"涌现/能力跳变"是评分器不连续性的伪影。如果你用连续指标（比如 cross-entropy），模型从来都是平滑改进。**写预算提案时，永远在连续指标上预测，不要在不连续指标上赌跳变。**

**关于"有效计算量"这个概念。** 这是 2026 年最被忽视的工具箱。你可能没听说过 Muon 优化器——它在匹配数据上比 AdamW 能多产出 2 倍的"有效运算"。同样的数据、同样的 GPU 小时、同样的架构——改个优化器，你训练出的模型等价于花了 2 倍 FLOPs。**在一个幂律世界里，常数因子以指数方式放大。一个 2 倍优化器增益相当于多花了 2^(1/α) ≈ 8 倍的计算量来推动损失下降。** 这是教科书上永远不会告诉你的"泄漏"式优化。

最后的清单——如果你在做训练预算决策：

- ✅ 你的 N/D 比例是多少？——如果 <5 就是 Kaplan 式训练不足，如果 >500 而你还要再训练更多代——检查是否在逼近 E 的损失地板。
- ✅ 你的推理预算占 TCO 多少？——如果超过 50%，D/N 应该远离 20，偏向更多数据。
- ✅ 你的训练数据质量够好吗？——数据筛选（Phi-style）可能给你 2-5 倍的有效计算提升——比一个更大模型划算。
- ✅ 你关注过优化器吗？——如果你还在用 AdamW 且没有特殊原因，试着问一句"Muon 行不行？"
- ✅ 你对"涌现"的期待是基于连续指标的预测还是基准跳变的传言？——在 roadmap 押注只会放大你的方差。

**金句：缩放定律不回答"这个模型好不好"，它回答的是"在你拥有的预算下，参数的堆砌有没有和数据的消耗对齐"——对齐了，算力没有浪费；没对齐，你在花钱犯方向错误。**

回到开头的问题——为什么你训练模型的方式可能是错的？因为你可能在分配两个 knobs 时压错了方向。Kaplan 们让你把 knob 全拧在"参数"上。Chinchilla 告诉你要平衡。Llama 告诉你——如果你考虑推理成本，过度训练那一边才是正确的。而这一切不是"优化"，是你做任何其他优化之前的"预算基础面"。基础面错了，上头堆再多技巧都是斜塔。
