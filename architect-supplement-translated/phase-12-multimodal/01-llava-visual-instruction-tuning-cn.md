# LLaVA 与视觉指令微调

> LLaVA（2023年4月）是全球被复制最多的多模态架构。它用2层MLP替换了BLIP-2的Q-Former，用朴素的令牌拼接替换了Flamingo的门控交叉注意力，并在GPT-4从纯文本标注生成的15.8万条视觉指令轮次上训练。2023到2026年间几乎所有构建VLM的从业者都是LLaVA的某种变体。LLaVA-1.5加入了AnyRes。LLaVA-NeXT提升了分辨率。LLaVA-OneVision在单一配方中统一了单图、多图和视频。这节课阅读配方，实现投影器，并解释为什么"更简单赢了"。

**类型：** 构建
**语言：** Python（标准库，投影器 + 指令模板构建器）
**前置条件：** 第12阶段 · 02（CLIP），第11阶段（LLM工程——指令微调）
**时间：** ~180分钟

## 学习目标

- 构建一个2层MLP投影器，将ViT图块嵌入（维度1024）映射到LLM的嵌入维度（维度4096）。
- 走通LLaVA的两阶段配方：（1）在55.8万标注对上做投影器对齐，（2）在15.8万GPT-4生成的轮次上做视觉指令微调。
- 构造LLaVA格式的提示，包含图像令牌占位符、系统提示和用户/助手轮次。
- 解释为什么社区从Q-Former转向MLP，尽管Q-Former在令牌预算上更优。

## 问题

BLIP-2的Q-Former（第12.03课）将一张图片压缩为32个令牌。干净、高效，基准测试表现好。但它有两个问题。

第一，Q-Former是可训练的，但其损失函数不是最终任务。第一阶段训练ITC+ITM+ITG。第二阶段训练LM损失。查询向量学习某种中间表示，然后LLM必须解码它。信息在瓶颈中丢失。

第二，Q-Former有1.88亿参数，在LLaVA 2023年的规模下，你必须为每个目标LLM协同设计它。换一个LLM，重训Q-Former。换一个视觉编码器，重训。每一种组合都是一个独立的R&D项目。

LLaVA的回答简单到令人尴尬：取出ViT的576个图块令牌，每个通过一个2层MLP（`1024 → 4096 → 4096`），然后把全部576个令牌直接扔进LLM的输入序列。没有瓶颈。没有用怪异目标做第一阶段预训练。仅用直接的LM损失训练MLP。

数据从哪里来？LLaVA的第二个洞见：用GPT-4（纯文本）生成指令数据。将COCO标注和边界框数据输入GPT-4，让它生成对话、描述和复杂推理问题。免费获得15.8万条指令-回应轮次。无需人工标注。

结果：一个在8张A100上跑一天就能训练出的VLM，在MMMU上击败Flamingo，并发布了社区可以扩展的公开检查点。到2023年底，它已经催生了50多个分支。

## 概念

### 架构

LLaVA-1.5，13B版：
- 视觉编码器：CLIP ViT-L/14 @ 336（第一阶段冻结，第二阶段可选解冻）。
- 投影器：2层MLP，带GELU激活，`1024 → 4096 → 4096`。
- LLM：Vicuna-13B（后来用Llama-3.1-8B）。

对一张图片 + 文本提示的前向传递：

```
img -> ViT -> 576个图块，维度1024
图块 -> MLP -> 576个令牌，维度4096
提示：system + "<image>" 占位符 + 用户问题
用576个投影令牌替换 <image> 令牌
将完整序列送入LLM
解码回应
```

图片占用LLM上下文的576个令牌。在2048上下文中，留给文本1472个令牌。在32k上下文中，只是一个舍入误差。

### 第一阶段：投影器对齐

冻结ViT。冻结LLM。仅训练2层MLP。数据集：55.8万图像-标注对（LAION-CC-SBU）。损失：以投影图像令牌为条件，对标注做语言建模。

以批次128跑一个epoch，几个小时就完成。投影器学会将ViT空间映射到LLM空间。不需要任务特定的监督。

### 第二阶段：视觉指令微调

解冻投影器（仍然可训练）。解冻LLM（通常完全解冻，有时用LoRA）。在15.8万条视觉指令轮次上训练。

指令数据是诀窍所在。Liu等人通过以下方式生成：
1. 取一张COCO图片。
2. 提取文本描述（5条人工标注 + 边界框列表）。
3. 用三种提示模板发送给GPT-4：
   - 对话："生成一个关于这张图的用户和助手之间来回对话。"
   - 详细描述："给出这张图的丰富、详细描述。"
   - 复杂推理："提出一个需要推理出图片内容的问题，然后回答它。"
4. 解析GPT-4的输出为（指令，回应）对。

这些数据完全不直接接触图片——只有文本描述。GPT-4会幻觉出合理的图像内容。有些噪声，但它有效：15.8万轮次足以解锁对话能力。

### 社区为何复制这个方案

- 没有需要调参的阶段一特定损失。从头到尾都是LM损失。
- 投影器几小时就能训练完，而不是几天。
- LLM可以替换（LLaVA-Llama2、LLaVA-Mistral、LLaVA-Llama3），只需重训投影器。
- 视觉指令数据管道使用GPT-4，为新领域重生成数据也很便宜。

### LLaVA-1.5 和 LLaVA-NeXT

LLaVA-1.5（2023年10月）加入：
- 学术任务数据（VQA、OKVQA、RefCOCO）混合到指令微调中。
- 更好的系统提示。
- 2048 → 32k上下文。

LLaVA-NeXT（2024年1月）加入：
- AnyRes：将高分辨率图像分割成2x2或1x3网格的336x336裁剪块，外加一个低分辨率全局缩略图。每个裁剪块变成576个令牌；每张图总计约2880个视觉令牌。OCR和图表任务显著提升。
- 更好的指令数据混合，加入ShareGPT4V（高质量GPT-4V标注）。
- 更强的基座LLM（Mistral-7B、Yi-34B）。

### LLaVA-OneVision

第12.08课深入讲解OneVision。简版：相同的投影器，但用课程学习训练，在一个模型中以共享的视觉令牌预算覆盖单图、多图和视频。

### 与Q-Former的对比

| | Q-Former (BLIP-2) | MLP (LLaVA) |
|---|---|---|
| 每图视觉令牌 | 32 | 576（基础）或2880（AnyRes） |
| 可训练参数 | 1.88亿 + LM | 4000万 + LM |
| 第一阶段损失 | ITC+ITM+ITG | 仅LM |
| 换LLM | 需要重训 | 小幅重训即可替换 |
| 多图 | 笨拙 | 自然（拼接） |
| 视频 | 笨拙 | 自然（逐帧拼接） |
| 令牌预算 | 小 | 大 |

MLP在简洁性和令牌灵活性上胜出。Q-Former在令牌预算上胜出。到2023年底，令牌预算不再是约束瓶颈（LLM上下文扩展到32k-128k+），简洁性统治了一切。

### 提示格式

```
A chat between a curious human and an artificial intelligence assistant. The assistant gives helpful, detailed, and polite answers to the human's questions. USER: <image> Describe this image in detail. ASSISTANT: The image shows ...
```

`<image>` 是一个占位符令牌。在分词之前，它被替换为576个视觉令牌（或AnyRes的2880个）。分词器看到的序列比训练时稍长，但LLM可以处理这个新颖的输入，因为第一阶段教会了它。

### 参数经济

LLaVA-1.5-7B分解：
- CLIP ViT-L/14 @ 336：3.03亿（第一阶段冻结，第二阶段常解冻）。
- 投影器（2个线性层）：~2200万可训练。
- Llama-7B：70亿。
- 总计：73亿参数。第二阶段可训练：70亿 + 2200万投影器。

第二阶段训练成本：8xA100上约20小时。这是关键数字——一天，一个节点，可复现。这就是LLaVA传播的原因。

## 使用

`code/main.py`实现：

1. 2层MLP投影器（玩具规模 dim 16 → 32 → 32）纯Python实现。
2. 提示构建管道：系统提示 + `<image>` 替换为N个投影令牌 + 用户轮次 + 助手生成占位符。
3. 可视化工具：展示576令牌视觉块在LLM上下文中的占比（2k / 32k / 128k上下文消耗百分比）。

## 📝 教师备课总结与读后感

### 一、文档整体评价
这是一份优秀的"架构师视角"LLaVA解读。文档没有停留在论文摘要层面，而是从"为什么这样设计"切入，抓住Q-Former→MLP替换、两阶段训练、数据生成这三条核心线索展开。技术深度足够但不过度学术化——每个关键决策都有"为什么"和数字支撑，非常适合教学中做决策对比分析。

### 二、知识结构梳理
- **基础层**：ViT图块嵌入概念、MLP投影器的线性代数本质、LM损失作为统一训练信号
- **模式层**：两阶段训练策略（冻结→解冻的节奏控制）、令牌拼接作为跨模态融合方式、GPT-4生成指令数据作为低成本标注方案
- **应用层**：AnyRes高分辨率分块的实际部署、LLM可替换性带来的生态扩散、576令牌在日益增长的上下文窗口中的边际成本递减

### 三、核心洞察
1. **"更简单赢了"不是运气**——MLP替代Q-Former之所以赢，是因为2023年上下文窗口(32k-128k)使令牌预算不再是瓶颈。关键洞见：架构选择必须与系统约束对齐。
2. **两阶段训练是成本最优化**——阶段一（几个小时）建立跨模态映射基础，阶段二（一天）注入对话能力。这不是学术洁癖，而是算力效率。
3. **GPT-4生成指令数据是"知识蒸馏的逆用"**——用更强的文本模型生成训练数据去训练多模态模型，本质上是把文本模型的世界知识迁移到视觉管道中。
4. **LLM可替换性是扩散的关键**——VLM层叠的模块化设计（冻结ViT→MLP桥梁→任意LLM）使LLaVA从一开始就是协议而非产品。
5. **576令牌在2k上下文中的占比(28%)是早期瓶颈，但在32k中(1.8%)是舍入误差**——架构决策的时间价值极其重要。
6. **标注到指令的"两步间接"有价值**——GPT-4看文本标注生成对话，而非直接看图片，这引入了噪声但也带来了多样性，类似数据增强的效果。

### 四、教学建议
1. **用可视化强化令牌预算概念**——让学生直观看到576/2880个视觉令牌在2k、32k、128k上下文中的占比饼图。概念一下就通。
2. **手算投影器参数量**——1024→4096→4096两个线性层 + GELU + bias，让学生算出~22M，再算占73亿的0.3%。数字记忆比概念记忆牢固。
3. **边讲边画对比图**——Q-Former（瓶颈→32令牌→LM解码）vs MLP（展开→576令牌→LLM直接吃），结构差异一目了然。
4. **做"GPT-4生成数据"的现场演示**——给一个COCO标注，现场让GPT-4生成对话，学生立刻理解数据管道不需要人工标注。
5. **引入Prismatic VLMs消融作为批判性思维训练**——让学生预测"去掉阶段一会怎样"，然后读消融结果验证，训练科学直觉。
6. **用LLaVA-1.5 → NeXT → OneVision的升级路径讲工程设计迭代**——每次升级解决一个具体瓶颈（分辨率→数据质量→模态统一），引导学生建立系统优化思维。
7. **布置"选你自己的LLM"练习**——让学生尝试LLaVA-Qwen、LLaVA-DeepSeek等组合，体会投影器重训的成本和模块化设计的威力。

### 五、值得补充的内容
1. **与更晚的VLM架构对比**——Qwen-VL的"三阶段训练"（与LLaVA两阶段对比）和InternVL的"动态分辨率 + 像素洗牌"方案，可以帮助学生理解LLaVA不是唯一的答案。
2. **视觉令牌压缩的复兴**——2025-2026年出现的token压缩方案（如FastV、SparseVLM）实际上在"回归"类似Q-Former的思维，但用更好的方式（层级剪枝、KV缓存优化）。这个"螺旋上升"值得讲。
3. **实际推理延迟数据**——576视觉令牌在H100上的prefill时间，以及AnyRes 2880令牌的实际延迟影响，让学生对"架构选型=成本选型"有直观感受。
4. **Open-LLaVA生态的故事**——从LLaVA发布到BakLLaVA、LLaVA-Med、LLaVA-Rad等50+分支，这是开源社区对"足够好的架构"的集体认可，可以作为生态扩散案例。

### 六、一句话总结
LLaVA证明了在多模态AI中，"足够好的简单方案"比"理论更优雅的复杂方案"更有生命力——前提是上下文窗口已经足够大，让简单方案的成本可以承受。

# 🎓 Agent 架构课：视觉-语言的对齐方式——为什么576个令牌比32个更好

你有没有想过一个问题：把一张图片塞进一个语言模型里，到底是把图片"翻译"成文字再喂进去好，还是直接把原始视觉信号扔进去好？

LLaVA给出的答案骇人听闻：把576个图块令牌一股脑扔进去。不做压缩，不做瓶颈，不做重参数化。就是扔。

**但这个答案是对的。** 我来告诉你为什么，以及这个选择教会了我们关于系统架构的一切。

---

## 问题本质：信息量与结构化的平衡

在LLaVA出现之前，业界的主流思路是Q-Former式的"压缩派"：用一个小网络把一张图片压缩成32个向量，然后让LLM去理解和生成。看似优雅——32个令牌对上下文窗口友好，结构清晰，论文漂亮。

但我在生产环境中见过太多Q-Former的坑。它在学术基准上分数不错，但一到真实场景就开始"说不清楚"。为什么？因为32个向量根本不够承载一张图片的全部信息量。一个VGA分辨率的图像有307200个像素——你用32个浮点向量去摘要它，信息损失率超过99.99%。

**我为什么在乎？** 因为我的生产系统每掉一个百分点的准确率，不是论文里的小数点，而是真金白银的用户投诉和业务损失。

## 两条路径：Q-Former vs MLP 直通

让我把两条路摆明说：

**路径A：Q-Former 压缩派**
- 图片→ViT→Q-Former(32令牌瓶颈)→LLM
- 1.88亿额外参数
- 三阶段训练（ITC+ITM+ITG，然后才是LM损失）
- 换LLM必须重训Q-Former
- 每图视觉令牌：32

**路径B：LLaVA MLP 直通派**
- 图片→ViT→MLP(576令牌展开)→LLM
- 2200万额外参数（是A的1/9）
- 从头到尾都是LM损失
- 换LLM只需重训MLP
- 每图视觉令牌：576

看起来A更"学术"，B更"粗暴"。但B在2023年赢了，而且赢得很彻底。为什么？

## 深入原理：系统性优势的叠加

MLP方案的优势不是某一项特别突出，而是四件事同时发生：

**第一，损失函数的统一性。** Q-Former需要分阶段训练，每个阶段的损失函数不同。这意味着你在阶段一优化的东西，阶段二可能根本不需要。你知道在生产系统中调试一个三段式训练管道有多痛苦吗？每个阶段的超参数互相影响，调A就得改B。LLaVA从头到尾就是"语言模型预测下一个词"，训练目标单一，调试简单。这不是理论优势——这是工程上一天的训练 vs 一周的调试。

**第二，信息完整性。** 576个令牌 vs 32个令牌，这是18倍的信息通道宽度。LLaVA-1.5的CLIP ViT-L/14输出的每个令牌都带着原图某一块的信息。当你把它们全部送入LLM，LLM的注意力机制自己决定哪些块重要、哪些不重要。你让模型自己做信息筛选，而不是用一个Q-Former替它筛选完之后，再让LLM去解码那个已经被筛选过的信息。这在OCR场景中尤其重要——Q-Former压缩后的小字信息直接丢失。

**第三，模块化设计。** LLaVA的架构天然是"三明治"：上面的ViT可以换（CLIP换SigLIP），中间的MLP管道很薄很便宜，下面的LLM可以随便换（Vicuna换Llama换Mistral换Qwen）。每次LLM出新版本，你只需要重新训练2200万个参数，几小时搞定。Q-Former则每次要重训1.88亿参数，并且设计空间是耦合的。我在生产环境中每三个月换一次基座LLM——如果每次都要重训Q-Former，我的GPU账单会让我被开除。

**第四，多图和视频的自然扩展。** 两张图怎么办？MLP派：各自投影576个令牌，拼一起1152个，送进去。视频呢？逐帧投影576个令牌，全拼起来。Q-Former派：你需要设计一个跨帧的压缩机制，又是一个新论文。简洁方案的自然扩展性是它最大的隐性价值。

## 生产数字：算力成本的真相

让我给你真实的数字。LLaVA-1.5-7B：

- 阶段一（投影器对齐）：558k标注对，批次128，几个小时。算力成本约$50-100。
- 阶段二（指令微调）：158k轮次，8×A100，20小时。算力成本约$400-600。
- **总计：一天一个节点，不到$700。**

对比BLIP-2（Q-Former路径）：阶段一单独就要几天，因为它有三个损失函数要同时优化，收敛速度慢。总成本至少翻倍。

但我最关心的不是绝对成本——是"换一个新的LLM需要多少钱"。

- LLaVA方案：重训MLP投影器，几个小时，$50-100。
- Q-Former方案：重训整个Q-Former，几天，$500-1000+。

当你的产品每季度需要尝试验证新LLM的时候，这个差距不是技术偏好，是商业可行性。

## 反模式：为什么还有人用压缩方案

不是说压缩方案都是错的。如果你在做一个上下文只有2k的嵌入设备，576个视觉令牌占了28%的上下文，压缩是必需的。但是——

**如果你在2025年还在为一个32k上下文的模型设计视觉令牌压缩，你在用一个2019年的问题拷打2025年的架构。**

上下文窗口的增长速度远超视觉令牌需求。LLaVA选576个令牌的时候，它占2k上下文的28%，是一个需要谨慎对待的设计选择。现在32k是标配，576令牌占1.8%，是一个舍入误差。**当资源不是瓶颈时，简洁永远优于聪明。**

还有人在设计VLM时执着于"降低视觉令牌数"，好像这是免费的午餐。不是免费的。每次压缩都在丢失信息。你降低了令牌数，然后惊讶地发现OCR准确率掉了15%，细粒度识别也差了。你省了1.5%的上下文，丢了15%的能力。这不是优化，这是自残。

## 结语：架构师的决策清单

设计一个VLM时，问你三个问题：

1. **我的最大上下文窗口是多少？** 如果≥16k，直接上MLP直通方案，不要纠结令牌数。
2. **我多久换一次LLM？** 如果每季度换，投资轻量投影器架构，别碰与LLM耦合的方案。
3. **我的数据预算多少？** 如果标注数据少，考虑"GPT-4生成指令数据"管道——你的文本模型就是你的标注器。

**金句：在资源充裕的时代，简洁性不是美德——它是生存策略。那个让你觉得"太简单了不可能work"的方案，通常就是那个work的。**

LLaVA的论文标题里有一个词值得反复咀嚼：Visual Instruction Tuning。不是Visual Representation Learning，不是Multimodal Fusion Academy，是"指令调优"。因为LLaVA的作者们看透了一件事——VLM的秘密不在于如何表示图像，而在于如何教LLM理解图像的语义。而教它的最好方式，就是给它所有的视觉信息，然后用语言监督去引导它。不多不少，刚刚好。
