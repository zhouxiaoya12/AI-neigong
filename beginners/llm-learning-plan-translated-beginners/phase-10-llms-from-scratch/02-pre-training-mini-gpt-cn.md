# 预训练一个 Mini GPT（1.24 亿参数）

> **原文：** Pre-training a Mini GPT (124M parameters)  
> **预计时间：** ~45 分钟  

---

## 🚗 开场翻车：你知道 Transformer 长什么样，但不知道它为什么做梦

99% 的 AI 工程师用预训练好的模型。GPT-4、Claude、Llama——下载权重，调 API，完事。

但当模型出问题——幻觉、自我重复、拒绝指令——你不知道为什么。因为你看过 Transformer 的图，但从没亲手让它跑过一个训练循环。

**GPT-2 Small 有 124,438,272 个参数。每一个都是训练循环里学出来的：前向→算损失→反向→更新权重。** 12 层 Transformer、12 个注意力头、768 维嵌入空间。每次模型吐一个 token，所有 1.24 亿个参数都在参与一条矩阵乘法链。

这节课的意义：**你知道原理，但只有亲手 build 过，你才知道为什么它会"疯"。**

---

## 🧠 GPT 架构——从 token ID 到下一个词的概率

```
token ID 进来 (batch, seq_len)
    ↓
Token Embedding: 每个 ID → 768维向量
    ↓
Position Embedding: 每个位置 → 768维向量
    ↓
相加
    ↓
Transformer Block × 12
    ↓
Layer Norm
    ↓
线性头 (768 → 50257 词汇量)
    ↓
Softmax → 下一个 token 的概率分布
```

### Transformer Block 内部

每个 Block：
1. **Layer Norm** — 稳定数值
2. **多头自注意力** — "这句话里哪个词跟哪个词相关？"
3. **残差连接** — 把输入加回去（防止梯度消失）
4. **Layer Norm**
5. **前馈网络** — "把每个位置的向量做一次非线性变换"
6. **残差连接**

重复 12 次。

### 注意力机制——为什么要"看上下文"

```
"The cat sat on the mat"
     ↑          ↑
    "cat" 需要知道 "sat" 是它的动词
    自注意力做的事：计算每对词之间的相关度
```

```python
# 简化的注意力计算
# Q=Query("我在查什么"), K=Key("我有什么"), V=Value("我的内容")
def attention(Q, K, V):
    scores = Q @ K.T / sqrt(d_k)  # 相关度分数
    weights = softmax(scores)      # 归一化
    return weights @ V             # 加权求和
```

## 🏆 焊死在脑子里的东西

1. **GPT = Embedding + 12 个 Transformer Block + 线性头 + Softmax。** 就这么简单。
2. **注意力 = "每个词看所有其他词，取相关部分的加权和"。**
3. **残差连接救了深度学习。** 没有它 12 层的梯度早没了。
4. **1.24 亿参数不是魔法——是 1.24 亿个数字，每个都是训练循环里调出来的。**
