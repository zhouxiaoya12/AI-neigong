# 贡献指南

感谢你对 AI 工程九阴真经的关注！

## 如何贡献

### 提 Issue

发现了错误、有改进建议、或者想讨论某个技术点？直接提 Issue。

好的 Issue 包含：
- **你看到的问题**：具体的文件名、行号、原文vs你认为正确的写法
- **你的背景**（可选但加分）：你是初学者还是有经验的工程师，这帮助我们判断问题的性质
- **建议的修改**（可选）：如果你知道怎么改，直接写出来

### 提 PR

欢迎 PR。提交前请：

1. **保持风格一致**：代码注释用中文，变量名保留英文
2. **代码必须能跑**：`python3 your-file.py` 零报错
3. **文章遵循现有格式**：技术深度 + 翻车案例 + 可运行代码
4. **一个 PR 做一件事**：修复错别字和新增一个章节分两次提

### 目前最需要的帮助

| 优先级 | 方向 | 说明 |
|--------|------|------|
| 🔴 最高 | 初学者友好版补充 | `beginners/` 目录下的文章需要更多中文用例和更平缓的学习曲线 |
| 🟡 重要 | 代码示例本地化 | 把依赖外部 API 的代码改成本地可跑的版本 |
| 🟡 重要 | 术语翻译讨论 | 中英术语对照是否需要调整？欢迎讨论 |
| 🟢 欢迎 | 新增章节 | 你觉得缺了哪个重要话题？提 Issue 先讨论再写 |
| 🟢 欢迎 | 错别字/格式修复 | 任何笔误都可以直接提 PR |

## 写作规范

- **深度优于广度**：一个概念往下挖三层，不要蜻蜓点水
- **工程直觉优先**：不讲"AI 是什么"，讲"AI 怎么工程化"
- **代码必可跑**：每一章的代码都能在本地跑通
- **中文友好**：注释用中文，示例用中文语境
