# 多 Agent 辩论与协作

> Du 等人（ICML 2024，"思想的社群"）运行 N 个模型实例独立提出答案，然后在 R 轮中迭代互相评审以收敛。提升事实性、规则遵循和推理。稀疏拓扑在令牌成本上优于全连通网状。

**类型：** 学习 + 构建
**语言：** Python（标准库）
**前置课程：** Phase 14 · 12（工作流模式）、Phase 14 · 05（Self-Refine 与 CRITIC）
**用时：** 约 60 分钟

## 学习目标

- 解释辩论协议：N 个提案者，R 轮，收敛到一个共享答案。
- 描述为什么辩论能提升事实性、规则遵循和推理。
- 解释稀疏拓扑：不是每个辩论者都需要看到每一个其他辩论者。
- 实现一个基于标准库的脚本化 LLM 辩论，包含全连通网状和稀疏变体；测量令牌成本 vs 准确性。

## 问题

Self-Refine（第 05 课）是一个模型自我评审——有群体思维风险。CRITIC（第 05 课）将评审锚定在外部工具上——并非总是可用。辩论引入了第三种模式：多个实例，交叉评审，通过分歧收敛。

## 概念

### 思想的社群（Du 等人，ICML 2024）

- N 个模型实例对同一问题独立提出答案。
- 在 R 轮中，每个模型阅读其他模型的提案并评审它们。
- 模型根据评审更新其答案。
- R 轮后，返回收敛的答案。

原始实验因成本使用 N=3, R=2。在困难问题上（MMLU、GSM8K、棋步合法性、传记生成），准确性随更多 Agent 和更多轮次提升。

跨模型组合优于单模型辩论：ChatGPT + Bard 一起 > 单独任何一个。

### 稀疏拓扑

"Improving Multi-Agent Debate with Sparse Communication Topology"（arXiv:2406.11776, 2024-2025）表明全连通网状辩论并不总是最优。稀疏拓扑（星形、环形、轴辐式）可以在更低令牌成本下匹配准确性。每个辩论者只看到同伴的子集。

含义：

- 全连通网状 N=5, R=3 = 5×3 = 15 个提案，每个读 4 个同伴 = 60 次评审操作。
- 星形 N=5, R=3（一个中心 + 4 个辐射） = 15 个提案，辐射只读中心 = 12 次评审操作。

### 辩论何时有帮助

- **事实性。** N 个独立提案，交叉检查减少幻觉。
- **规则遵循。** 棋步合法性——一个模型漏了一条规则，其他模型捕获。
- **开放式推理。** 多个框架收敛到正确答案。

### 辩论何时有害

- **延迟敏感 UX。** N×R 轮串行是你可能没有的延迟。
- **成本敏感规模。** 每个问题 N×R 个令牌。
- **简单事实查询。** 一次查询比五次辩论便宜。

### 2026 年实践实例化

- **Anthropic 编排器-工作者**（第 12 课）——带有综合步骤的辩论变体。
- **LangGraph 监管者**（第 13 课）——中央路由器 + 专家 Agent 可将辩论实现为一个节点。
- **OpenAI Agents SDK**（第 16 课）——Agent 来回转交进行迭代评审。
- **多 Agent 评估**——将辩论 + 评估器-优化器配对用于评估信号。

### 此模式在何处会出错

- **收敛坍缩。** 所有 Agent 收敛到第一个错误答案。用强制分歧轮次缓解。
- **中心故障。** 在星形拓扑中，一个坏中心污染所有人。轮换或使用多个中心。
- **提示同质化。** 所有 Agent 使用相同提示；它们产出相同的答案。使用多样化的提示和/或模型。

## 📝 教师备课总结与读后感

### 一、文档整体评价

这是 Phase 16 高级 Agent 工程中的"集体智能"课。前面五节课讲的是 Agent 作为个体系统——它怎么跑、怎么不崩、怎么省钱、怎么由人类监督。这节课是 Agent 作为群体——多个模型互相辩论以产出比任何单个更好的答案。核心洞察是反直觉的：**不是"一起思考更好"——是"分歧更好"。** 辩论的价值在于不同观点产生交叉检查——而稀疏拓扑的价值在于"不全量通信"能匹配全通信的准确性且更便宜。适合已理解 Self-Refine/CRITIC、正在处理高可靠性需求（医疗、法律、金融）的架构师。

### 二、知识结构梳理

**辩论协议层：** N 个提案者独立提出答案 → R 轮交叉评审 → 根据评审更新答案 → 收敛。这是多 Agent Self-Refine——区别在于评审者不是"自己的另一面"而是"不同的模型/提示/视角。"

**拓扑优化层：** 全连通网状（每个辩论者每轮读所有同伴）vs 稀疏（星形/环形/轴辐式）。稀疏在更低令牌成本下匹配全连通网状的准确性——关键原因是"不需要所有辩论者看到所有其他辩论者；评论的多样性在某个点饱和。"

**工程实例化层：** 三个主流框架的本课模式实现——Anthropic 编排器-工作者、LangGraph 监管者节点、OpenAI Agents SDK 转交。

### 三、核心洞察（备课时的关键理解）

1. **辩论和 Self-Refine 的关键差异是"评审者不是自己。"** Self-Refine：一个模型写，同一个模型评。→ 群体思维风险。辩论：模型 A 写，模型 B 和 C 评。→ 不同视角的交叉检查。本质上，辩论用多个模型之间的分歧作为信号，Self-Refine 只有一个模型内部的信号。

2. **跨模型组合（ChatGPT + Bard）优于单一模型辩论，因为差异性是辩论的价值来源。** 两个相同的 GPT-4 实例辩论——它们产出相似的推理路径，交叉检查的价值低。GPT-4 + Claude 辩论——不同的训练数据、不同的推理风格——交叉检查的价值高。辩论的准确性收益来自评审者的差异性。

3. **稀疏拓扑的发现（"不全量通信也能一样好"）是成本-准确性 trade-off 的直接优化。** 全连通网状 N=5, R=3 = 每个辩论者读 4 个同伴 = 大量令牌。星形 N=5 = 辐射只读中心 = 大幅减少令牌。关键是——中心积累了所有信息，辐射从中心获得的是"综合的"反馈而非"每一个同伴的原始输出"。信息经过压缩但保留了批判性价值。

4. **"强制分歧"规则是防止辩论堕落为"所有人同意第一个错误答案"的机制。** 在第一轮，强制每个辩论者产出不同的提案。如果无法产生不同的提案→分歧是假的，辩论没有意义。这是元评估——辩论协议需要验证参与者是否真正持有不同视角。

5. **辩论是评估器-优化器模式的扩展——用 N 个评估器代替一个。** CRITIC：一个外部工具做验证。辩论：N 个其他模型做验证。当外部工具不可用时（开放式推理、创意评估、伦理判断），辩论是 CRITIC 的替代方案。

6. **收敛坍缩（所有 Agent 迅速同意第一个错误答案）是辩论最危险的失败模式。** 当所有 Agent 共享相似的训练数据、相似的推理路径——第二个 Agent 看到第一个的答案说"有道理"，第三个看到前两个都同意也说"有道理"。缓解：① 强制分歧轮次 ② 不同的基础模型 ③ 不同的提示风格。

7. **星形拓扑的中心故障是系统级的单点故障。** 如果中心辩论者产生了一个错误但表达得非常自信的答案——所有辐射可能跟着错。缓解：轮换中心角色（每轮不同）、使用多中心、对中心的输出做额外的 CRITIC 验证。

### 四、教学建议

1. **用"三个医生看同一张 X 光片"的类比开场。** 医生 A 说"骨折"，医生 B 说"只是挫伤"，医生 C 说"骨折"。三人讨论——B 改变意见。这不是"多数投票"——是"基于他人推理改变自己意见。"这就是辩论协议。

2. **Self-Refine→CRITIC→Debate 是一个自然的演进。** Self-Refine：自己评自己。CRITIC：外部工具评自己。Debate：多个"他者"互相评。三者是同一个问题的三种答案：**"谁来确保输出是对的？"**

3. **稀疏拓扑的节省用一个数字 demo 来讲。** 全连通网状 N=5：每个辩论者产生一次提案 + 读 4 次他人的 = 5×5 = 25 次操作/轮。星形 N=5：5 次提案 + 4 次读中心 = 9 次操作/轮。学生算一下——然后看准确性对比。这是"用数学做架构决策"的范例。

4. **收敛坍缩用一个坏的 demo 来展示。** 三个相同的 GPT-4-mini 实例，同一个提示。问题："这段代码有 bug 吗？" Agent 1 说"没有。" Agent 2 和 3 都说"我也觉得没有。"辩论 1 轮就收敛了——全部错了。然后换成两种不同模型——Claude Haiku 说"有一个边界条件错误"。辩论真正发生了。

5. **跨模型组合的准确性收益用一个对比实验。** 3 个 GPT-4o vs 1 GPT-4o + 1 Claude Sonnet + 1 Gemini Flash。在规则遵循任务上（棋步合法性），异构组通常更好。让学生看到"差异性 = 价值"。

6. **练习 3（异质性实验）是最诚实的。** 让学生换一个有不同"意见"的脚本化 LLM。异构组的准确性是否比同构组更好？不是假设——是数据。

7. **练习 4（成本 vs 准确性图）是架构师的核心技能。** 让学生画一张散点图：X 轴 = 令牌成本，Y 轴 = 准确性。全连通网状 N=3 一个点，全连通 N=5 一个点，星形 N=5 一个点。哪个点在 Pareto 前沿上？这是选型决策——不是"哪个更好"，是"你的预算和准确性需求的交点在哪。"

### 五、值得补充的内容

1. **辩论中的"少数派保护"——当 4 个 Agent 错了，1 个对了时。** 多数投票会淹没正确但非共识的观点。需要异议报告机制——不压制少数派，记录并标记。

2. **辩论的延迟模型。** 全连通网状 N=5, R=3 = 每轮 5 次并行提案 + 然后串行评审依赖。实际墙钟时间取决于并行化程度——全并行 vs 半并行 vs 全串行。

3. **与监管者模式（第 17 课）的对比。** 监管者 = 有一个 Agent 做最终综合决策。辩论 = 无中心决策者，通过迭代分歧自然收敛。监管者适合"任务分配"，辩论适合"答案验证"。

4. **辩论在非标准任务上的应用。** 代码审查、创意写作评审、策略规划——这些不是"有一个正确答案"的领域——辩论需要不同的收敛标准。

5. **辩论中的"退出条件"——什么时候停止？** 收敛（所有 Agent 同意）vs 最大轮次 vs 分歧不可调和（Agent 之间有根本性分歧，没有人在改变意见）→ 升级给人类。

### 六、一句话总结

**多 Agent 辩论的秘密不在"一起思考更好"——在"不同的思考者互相挑战时，错误更容易被暴露。"一个模型可能同意自己的错误。两个相同的模型可能互相加强错误。两个不同的模型在暴露彼此的错误——这不是因为任何单个 Agent 更聪明，是因为分歧本身就是一个验证信号。**

# 🎓 Agent 架构课：多 Agent 辩论——为什么分歧比一致更有用

同学们好。我是你们的 Agent 架构老师。

前面五节课我讲了 Agent 作为个体——怎么跑长程、怎么监管、怎么持久化、怎么预算、怎么由人类监督。今天这节课不一样。今天讲的是**当你不只一个 Agent 时——你让它们互相辩论，然后答案比任何一个单独的都好。**

你旁边有三个同事。你问他们同一个问题。同事 A 说答案应该是 X。同事 B 看了 A 的推理，说"等一下，你的第二步有漏洞——所以我觉得是 Y。"同事 C 读了 A 和 B 的推理，说"B 说得对，但我发现第三步也不对——应该是 Z。"你听到了全部，你自己的判断比只看一个人的答案准了很多。

这不是你在做"多数投票"。是**不同的头脑交叉检查对方的推理，错误在这个过程中暴露。** 这就是多 Agent 辩论。

## Self-Refine → CRITIC → Debate：同一问题的三种答案

这堂课是前面两节课的直接延续。

第 02 课讲了 Self-Refine：一个模型写、同一个模型改。问题是什么？**模型可能同意自己的错误。** 它检查自己的输出——"看起来没问题。"——因为"看起来没问题"和"真的没问题"在同一个模型的内部逻辑里很难区分。

第 02 课也讲了 CRITIC：外部工具做验证。跑测试、查数据库、搜事实。问题是什么？**不是所有任务都有外部工具。** 伦理判断、创意写作、策略分析——没有"正确答案 API"可以调。

辩论是第三种答案：**不需要外部工具——但有多个视角。** 不是"我问我自己有没有错"——是"我让你的同事问你有没有错。"

## 为什么差异性最重要

Du 等人的实验里有一个特别关键的发现：**ChatGPT + Bard 一起辩论，比两个 ChatGPT 辩论更准。**

为什么？因为辩论的价值来自**评审者的差异性。** 两个相同的模型看到相同的问题，产生相似的推理路径。相同的盲点。它们同意对方是因为它们想得一样。

不同的模型有不同的训练数据、不同的推理风格、不同的知识边界。Claude 可能在某个领域比 GPT 强，GPT 在另一个领域更强。放在一起辩论——一个人漏的另一个人可能捕获。

2026 年的实践中，异构辩论（不同模型 + 不同提示）是默认选择——不只是"多个 Agent"，是"多种 Agent。"

## 稀疏拓扑：不是所有人都需要读所有人

全连通网状辩论——N=5，每个 Agent 每轮读其他 4 个的提案。5 个 Agent × 3 轮 = 每个 Agent 产生 3 次提案 + 读 12 次他人的提案 = 大量的令牌。

一篇 2024-2025 年的论文提出了一个反直觉的发现：**不需要所有人都读所有人。** 星形拓扑（一个中心，四个辐射只读中心）在更低令牌成本下匹配准确性。环形拓扑（每个人只读左右邻居）也是。

为什么？因为评审信息的价值在某个点饱和。Agent A 读了 Agent B 的评审——这有价值。Agent A 再读 Agent C 的评审——C 的评审可能和 B 的类似——边际价值下降。稀疏拓扑保留了足够的批判性多样性但砍掉了冗余通信。

对工程来说：**全连通是默认，稀疏是优化。** 从全连通起步。当成本成为问题时——切到稀疏。

## 什么时候辩论会崩

**收敛坍缩。** Agent 1 说"答案是 X。" Agent 2 看了——"有道理，我也觉得是 X。" Agent 3 看了前两个都同意——"那我也觉得是 X。"第一轮就收敛了——全错了。

为什么会这样？因为三个 Agent 想得太像。相同的模型、相同的提示、相同的知识边界。辩论从来没有发生过——只发生了"快速同意。"

修复：① 强制分歧轮次——第一轮，每个 Agent 必须产出不同的提案。如果无法产生不同的提案，辩论没有意义。② 不同模型——Claude + GPT + Gemini。③ 不同提示——"你是激进派"/"你是保守派"/"你是怀疑论者。"

**中心故障。** 星形拓扑——中心 Agent 在第一轮产生了一个错误但非常自信的答案。四个辐射 Agent 读了这个答案——中心听起来很权威——全部跟着错。轮换中心角色，使用多中心。

## 辩论的成本数学

N=3, R=2, 全连通 = 每个辩论者读 2 个同伴 = 3×2 = 6 次评审操作/轮 × 2 轮 = 12 次。加上 3×2 = 6 次提案 → 总共 18 次 LLM 操作。

对比单次 Self-Refine：1 次生成 + 1 次评审 + 1 次修改 = 3 次。

辩论是 6 倍的成本。**你只在错误的代价比辩论的成本高时才用辩论。** 医疗诊断：错误 = 误诊，辩论的成本 = $0.50。法律建议：错误 = 法律责任，辩论的成本 = $2。这些场景——辩论是廉价保险。客服回复：错误 = "抱歉，我理解错了"，辩论的成本 = $0.30。这些场景——不需要辩论。

## 结课清单

实现多 Agent 辩论：

1. N 个提案者、R 轮交叉评审——收敛到一个答案。
2. 异构模型 > 同构模型——差异性 = 批判性价值。
3. 全连通起步，稀疏优化——成本-准确性 trade-off。
4. 强制分歧轮次——防止第一轮快速同意的坍缩。
5. 星形拓扑中心轮换——防止中心故障污染所有 Agent。
6. 只在错误代价 > 辩论成本时用它——不是每个任务都值得 6 倍令牌。

**最后一句话：**

**多 Agent 辩论给你的不是"更聪明的 Agent"——是 Agent 之间的不一致成为一个信号。当三个不同的模型对同一问题得出不同答案时，它们的差异在交叉检查中暴露了各自的盲点。辩论不靠任何人更聪明——靠没有人能同时骗过所有人，如果"所有人"不是同一个模型的三个副本。**
