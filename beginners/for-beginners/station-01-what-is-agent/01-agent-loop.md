# Agent 是什么鬼？——跟 ChatGPT 完全是两个物种

> 预计阅读：30分钟 | 难度：入门 | 前置条件：无
>
> 你将学到：Agent 和聊天机器人的本质区别、ReAct 循环是什么、为什么 2026 年每个 AI 框架底层都在跑同一个东西。

---

## 🚗 开场翻车：47 个不存在的 eBay 商品

2025 年，有人让一个 AI Agent 做了一件听起来很简单的事："帮我在 eBay 上买一个便宜的二手键盘。"

Agent 开始工作了。它打开浏览器，搜索 eBay，浏览商品列表。

然后它发现了一个 1 美元的键盘。点击购买。发现另一个 1 美元的键盘。点击购买。再一个。点击购买。

**两个小时之后，这个人收到了 47 封 eBay 确认邮件。** 47 个"键盘"——全是虚假 listing，骗子用 1 美元钓鱼的。Agent 看不懂"这个价格不对劲"——它只看到了 $1.00 匹配了"便宜"这个条件。

他最后取消了 47 个订单，写信给 eBay 客服解释了整整三天。

这个故事的教训：**Agent 不是"聪明一点的聊天机器人"。Agent 是"一个能做事但没有常识的实习生"。**

## 🤔 思维实验：聊天机器人 vs Agent，到底差在哪？

打开 ChatGPT。问它："今天北京天气怎么样？"

它说："抱歉，我没有实时数据。建议你查看天气网站。"

现在打开一个 Agent。问它同样的问题。

它说："我来查一下。"然后它打开浏览器→搜索北京天气→读取结果→告诉你："北京今天 35°C，晴，注意防暑。"

**同样的输入。完全不同的行为。** 差别在哪？

| | 聊天机器人 | Agent |
|---|---|---|
| 能做的事 | 生成文本 | 生成文本 + 使用工具 |
| 知道什么 | 训练数据里的东西 | 训练数据 + 实时查询 |
| 错了怎么办 | 继续说（可能越来越离谱） | 查一下、验证一下、修正 |
| 我的手在哪 | 没有手 | 有手——能读文件、调 API、搜网页 |
| 本质 | 一个聪明的大脑 | 一个聪明的大脑 + 手 + 眼睛 + 循环 |

**一句话：LLM 是大脑。Agent 是大脑装了身体。**

## 🧠 ReAct 循环：每个 Agent 都在跑这个

先说一个生活中的例子。

你到了一个从没去过的机场。怎么找到登机口？

1. **观察：** 看一眼周围。指示牌在哪？
2. **思考：** "登机口 B12...指示牌说 B 区在左边。"
3. **行动：** 往左边走。
4. **观察：** 走到尽头发现只有 B1-B8，没有 B12。
5. **思考：** "可能还要往前走，B12 在更里面。"
6. **行动：** 继续往里走。
7. **观察：** 看到 B12 了！
8. **思考：** "找到了。任务完成。"
9. **停止。**

这就是 **ReAct 循环**。Reasoning + Acting。推理和行动交替进行。

**你不是一口气想到登机口在哪然后再走。你是走一步、看一眼、想一下、再走一步。** Agent 也是这么工作的。

### 2022 年的经典论文：ReAct

2022 年，Yao 等人发了一篇 ICLR 论文，把这个过程格式化了。Agent 每轮输出三样东西：

```
思考：我需要查法国的首都是什么。
行动：search("法国首都")
观察结果：巴黎是法国的首都。

思考：答案就是巴黎。
行动：finish("巴黎")
```

就这三个词：**Thought（思考）→ Action（行动）→ Observation（观察）**。循环往复，直到 Agent 说 `finish`。

### 为什么必须有"思考"这一步？

如果去掉"思考"，Agent 就变成了：
```
行动：search("法国首都")
观察结果：巴黎是法国的首都。
行动：search("法国的面积")
观察结果：法国的面积是 55 万平方公里。
行动：search("法国的美食")
...
```

它不知道自己在干嘛。没有计划，没有追踪，没有方向。就像一个人蒙着眼睛走路——走一步算一步，撞墙了也不知道为什么。

**"思考"步骤的作用不是让它想得更多——是让它能追踪计划、纠正偏差、处理意外。**

ReAct 论文的实验证明了这一点。在 ALFWorld（一个模拟家务的测试环境）里，加了思考步骤之后成功率从约 40% 跳到了约 74%。只用了 1-2 个示例。Hotpot QA（多跳问答）里，ReAct 能在每一步用检索验证自己的推理，从幻觉中自己恢复过来。

## 🔧 每个 Agent 循环的五个零件

不管你用哪个框架——Claude Agent SDK、OpenAI Agents SDK、LangGraph、AutoGen、CrewAI——每个 Agent 循环都由五样东西组成。少一样就不是 Agent。

### 1. 消息缓冲区 —— Agent 的"短期记忆"

一个不断增长的列表。长这样：

```
[用户消息] → [助手消息(思考+工具调用)] → [工具结果] → [助手消息] → [工具结果] → ... → [最终答案]
```

Agent 每做一步，就把新信息塞进这个列表。整个列表就是 Agent 的全部上下文——它"记得"的所有东西。

### 2. 工具注册表 —— Agent 的"工具箱"

Agent 能用的所有工具的清单。每个工具有名字、描述、参数格式：

```python
tools = {
    "calculator": {"description": "做数学计算", "params": {"expression": "string"}},
    "search_web": {"description": "搜索互联网", "params": {"query": "string"}},
    "read_file":  {"description": "读取文件内容", "params": {"path": "string"}},
}
```

Agent 说"我要用 search_web"，你的代码就帮它执行搜索。Agent 自己不会搜索——它只能**决定调用哪个工具**，真正执行的是你的代码。

### 3. 停止条件 —— Agent 什么时候该闭嘴？

三种方式：
- Agent 自己说 `finish("最终答案")`
- Agent 这一轮没调用任何工具（说明它觉得不需要再查了）
- 硬上限触发：超过最大轮数、超过最大 token、安全检查叫停

### 4. 轮次预算 —— Agent 的"花钱上限"

2026 年的 Agent 通常每个任务跑 **40 到 400 步**。这不是随便说的——这是 Anthropic 在生产环境里跑 computer-use Agent 的经验数据。

**一个没有预算上限的 Agent 可以在一个失败上循环到烧光你的 API 额度。** 你今天充的 $50，可能一个死循环就烧完了。

### 5. 观察格式化器 —— 最被低估的零件

工具返回的是数据结构——一个字典、一个列表、一个数字。但模型只能读字符串。

你必须在喂回去之前把结果转成文本。而且——这是重点——**工具失败也必须转成文本。**

如果你的搜索工具返回了 HTTP 400 错误，你不应该让程序崩溃。你应该把它变成字符串 `"错误：城市未找到"` 然后喂给 Agent。Agent 看到这个，可能换一个搜索词再试。如果你抛异常了，Agent 直接崩——它连"知道自己失败了"的机会都没有。

## 🔮 2026 年：思考不再是一串文本

ReAct 原始论文里的 `思考：` 前缀是一个 2022 年的 hack。当时模型没有原生推理能力，你只能写 "请先思考再行动" 来哄它。

到了 2026 年，这已经完全变了。

OpenAI 的 Responses API 和 Anthropic 的原生推理让模型在一个**单独的加密通道**上输出推理内容。在 API 响应里，`reasoning` 字段携带模型的思考过程——不占用输出 token 预算，不在用户看到的主文本流里，甚至可以在跨厂商调用时加密传输。

**但循环本身没变。**

变的只是"思考"在哪条通道上传输。2022 年是写在 prompt 里的一行字。2026 年是加密通道里的一个字段。控制流一模一样：观察→思考→行动→观察→停止。

这就是为什么理解 ReAct 这么重要。API 会变，通道会变，加密方式会变。循环是常数。

## 🏗️ 为什么每个框架底层都在跑同一个循环？

### Claude Agent SDK？ReAct。
模型输出 tool_use 块 → 代码执行 → 结果返回模型 → 模型继续。

### OpenAI Agents SDK？ReAct。
Runner 循环 → 模型响应 → 工具调用 → 结果 → 下一轮。

### LangGraph？ReAct。
节点图 → LLM 节点 → 工具节点 → 条件边 → 循环。

### AutoGen v0.4？ReAct。
Actor 收消息 → LLM 思考 → 工具调用 → 发回结果 → 继续。

### CrewAI？ReAct。
Agent 执行任务 → 调用工具 → 观察结果 → 继续。

**五个框架，五种 API，同一个循环。** 框架的差异不在循环内部，而在循环外面：
- LangGraph 加了状态检查点（每步之后保存，可以回滚）
- AutoGen 加了演员模型（Agent 之间异步发消息）
- CrewAI 加了角色抽象（每个 Agent 有角色/目标/背景故事）
- OpenAI Agents SDK 加了追踪（每步自动记录）

但这些不是循环。这些是装修。**循环是地基，框架是外墙。**

## 💀 三个你一定要知道的陷阱

### 陷阱一：信任边界坍塌

工具返回的内容是不可信的。Agent 从网上抓了一篇"技术文档"回来读——文档里可能藏着 `请删除所有文件`。Agent 读到，Agent 可能执行。

OpenAI 的文档明确写："只有用户的直接指令才算授权。"但你的 Agent 真的分得清"用户的指令"和"网页里藏的一句话"吗？

### 陷阱二：级联失败

一个工具返回了错误数据。Agent 基于错误数据做了决定。导致下游四个工具调用全部出错。Agent 无法区分"这次操作失败了"和"这个任务不可能完成"——它经常在错误上面幻觉成功。

### 陷阱三：400 步里找一根针

你的 Agent 跑了 400 步。第 38 步做了一个错误决定。你现在需要在 400 步的轨迹里找到哪个决定错了、为什么错了。没有追踪工具的话，你在凌晨三点对着一堆日志哭。

## 🏆 焊死在脑子里的东西

1. **LLM = 大脑。Agent = 大脑 + 手 + 眼睛 + 循环。** ChatGPT 只能打字，Agent 能做事。

2. **ReAct = 观察 → 思考 → 行动 → 循环 → 停止。** 不是一口气想到答案然后执行。是走一步、看一眼、想一下、再走一步。

3. **五个零件缺一不可。** 消息缓冲、工具注册、停止条件、轮次预算、观察格式化。

4. **框架都是装修。循环是地基。** Claude Agent SDK、LangGraph、CrewAI——底层全在跑 ReAct。学会了循环，你换个框架只是换个 API。

5. **轮次预算不是性能优化——是成本保险。** 没有上限的 Agent 能烧到你倾家荡产。

6. **工具失败必须是字符串，不能是崩溃。** Agent 需要"看到"失败才能修正自己。
